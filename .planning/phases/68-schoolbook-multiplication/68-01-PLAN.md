Create `c_backend/src/ToffoliMultiplication.c` with two functions:

**1. `void toffoli_mul_qq(circuit_t *circ, const unsigned int *ret_qubits, int ret_bits, const unsigned int *self_qubits, int self_bits, const unsigned int *other_qubits, int other_bits)`**

Implements schoolbook QQ multiplication: `ret = self * other` using shift-and-add with controlled CDKM adders.

Algorithm: For j = 0 to n-1, perform a controlled addition of `self[0..n-1-j]` (shifted by j) into `ret[j..n-1]`, controlled by `other[j]`:
- `width = n - j` for each iteration (progressively smaller additions)
- For width == 1: Build tqa as `[ret[n-1], self[0], other[j]]` (3 qubits), call `toffoli_cQQ_add(1)`, no ancilla needed
- For width >= 2: Allocate 1 carry ancilla via `allocator_alloc(circ->allocator, 1, true)`. Build tqa with CDKM convention (register swap per hot_path_add.c pattern):
  - `tqa[0..width-1]` = `self_qubits[0..width-1]` (a-register = source, preserved)
  - `tqa[width..2*width-1]` = `ret_qubits[j..j+width-1]` (b-register = target, gets partial sum)
  - `tqa[2*width]` = ancilla carry
  - `tqa[2*width+1]` = `other_qubits[j]` (control qubit = multiplier bit j)
  - Call `toffoli_cQQ_add(width)`, then `allocator_free` the ancilla
- Use `n = ret_bits` (result width). NULL-check all sequence returns.
- Use `unsigned int tqa[256]` on the stack (same pattern as hot_path_add.c).

**2. `void toffoli_mul_cq(circuit_t *circ, const unsigned int *ret_qubits, int ret_bits, const unsigned int *self_qubits, int self_bits, int64_t classical_value)`**

Implements schoolbook CQ multiplication: `ret = self * classical_value` using shift-and-add with uncontrolled CDKM adders.

Algorithm: Decompose classical_value into binary using `two_complement(classical_value, n)`. For each bit j where `bin[n-1-j] == 1` (LSB-first iteration), perform an uncontrolled addition of `self[0..n-1-j]` into `ret[j..n-1]`:
- `width = n - j` for each set bit
- For width == 1: Build tqa as `[ret[n-1], self[0]]` (2 qubits for CNOT), call `toffoli_QQ_add(1)`, no ancilla
- For width >= 2: Allocate 1 carry ancilla. Build tqa:
  - `tqa[0..width-1]` = `self_qubits[0..width-1]` (a-register = source, preserved)
  - `tqa[width..2*width-1]` = `ret_qubits[j..j+width-1]` (b-register = target, gets partial sum)
  - `tqa[2*width]` = ancilla carry
  - Call `toffoli_QQ_add(width)`, then `allocator_free` the ancilla
- Free the `bin` array at end. NULL-check all sequence returns. Clean up (free bin, free ancilla) on early return.

**Includes needed:** `"toffoli_arithmetic_ops.h"`, `"execution.h"`, `"qubit_allocator.h"`, `"Integer.h"` (for `two_complement`), `"QPU.h"`.

**Header update:** Add declarations for `toffoli_mul_qq` and `toffoli_mul_cq` to `c_backend/include/toffoli_arithmetic_ops.h` (after the existing addition declarations, before `#endif`).

**setup.py:** Add `os.path.join(PROJECT_ROOT, "c_backend", "src", "ToffoliMultiplication.c")` to the C source files list, right after the `ToffoliAddition.c` entry (around line 41-42).

IMPORTANT: The register swap convention must match hot_path_add.c exactly. The CDKM adder modifies the b-register (positions [bits..2*bits-1]). The a-register (positions [0..bits-1]) is preserved. So `self` (multiplicand, preserved) goes to a-register positions, `ret` (accumulator, modified) goes to b-register positions.
