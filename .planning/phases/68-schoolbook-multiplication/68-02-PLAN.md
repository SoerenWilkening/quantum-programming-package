**Verification helper `_verify_toffoli_mul_qq`:**
- Follow the pattern of `_verify_toffoli_qq` from test_toffoli_addition.py
- For QQ mul, the result register `ret` is at the FIRST position in the qubit array (positions [0..ret_bits-1]), per the hot_path_mul.c layout. The self register follows at [ret_bits..ret_bits+self_bits-1], then other at [ret_bits+self_bits..].
- But the Toffoli path builds its OWN qubit array internally and does NOT use the hot_path_mul.c qubit array layout. The ret_qubits are passed separately. In the Python API, `c = a * b` allocates a new result register. The physical qubit ordering in the circuit is: a qubits first (self), then b qubits (other), then c qubits (ret), then any ancilla.
- To extract the result, examine the QASM output. The result register (ret/c) was allocated AFTER a and b. For width w: a at [0..w-1], b at [w..2w-1], ret at [2w..3w-1], ancilla above that.
- Extract result from physical qubit positions [2*width .. 3*width-1] using `_simulate_and_extract` (same as test_toffoli_addition.py).
- Actually, the simplest approach: copy the `_simulate_and_extract` helper from test_toffoli_addition.py, then write `_verify_toffoli_mul_qq` that does gc.collect(), ql.circuit(), calls circuit_builder, exports QASM, parses qubit count, and extracts result from positions [2*width..3*width-1] (since ret is the 3rd register allocated).

**Verification helper `_verify_toffoli_mul_cq`:**
- For CQ mul (`qa *= val`), the result is in the ret register. Since `__imul__` allocates a new result and then swaps qubit references, the result is at [width..2*width-1] (ret allocated after self). But actually, `multiplication_inplace` creates a `ret` qint and fills it with the product. For `__imul__`, after the call, self's qubits are swapped with ret's qubits.
- In practice, for CQ imul: self qubits at [0..w-1], ret qubits at [w..2w-1], ancilla above. After the operation, qa's Python object points to ret qubits [w..2w-1]. The result should be extracted from [w..2w-1].
- Alternatively, use the simpler approach: build circuit with `qa *= val`, then extract from the positions where the result register is. Test by printing QASM for a simple case first, then hard-code the extraction.
- ACTUALLY, the safest approach (matching test_mul.py pattern): use the `verify_circuit` fixture from conftest.py. This fixture already handles the extraction correctly for the existing QFT multiplication tests. Look at how test_mul.py uses it: `verify_circuit(circuit_builder, width)` where circuit_builder returns the expected value. The fixture extracts the first `width` bits of the measured bitstring.
- HOWEVER, the conftest `verify_circuit` fixture extracts result from position 0 (first `width` bits of bitstring), which works for in-place operations where self is the result. For QQ mul where ret is a separate register, this won't work.
- DECISION: Write custom verification helpers (like test_toffoli_addition.py does) rather than using the conftest fixture. This gives full control over result extraction positions.

**Class `TestToffoliQQMultiplication`:**

- Call `_verify_toffoli_mul_qq(circuit_builder, width)`, collect failures, assert at end
- Expected: `(a * b) % (2^w)` (unsigned modular multiplication)

**Class `TestToffoliCQMultiplication`:**
- `test_cq_mul_exhaustive(self, width)` parametrized with widths [1, 2, 3]
- For each width, iterate all (a, b) pairs
- circuit_builder: `_enable_toffoli()`, create `qa = ql.qint(a, width=w)`, `qa *= b`, return `(a * b) % (1 << w)`
- Call `_verify_toffoli_mul_cq(circuit_builder, width)`, collect failures, assert at end
- For CQ mul, the result register is a NEW register allocated by multiplication_inplace. After `__imul__`, qa's qubits point to the result register. The self qubits are at [0..w-1], the ret qubits at [w..2w-1]. After imul swap, qa points to ret at [w..2w-1]. So extract result from [w..2w-1].
- WAIT: Actually re-examine. For `qa *= b` (int), `__imul__` calls `__mul__` which creates a new result register, then swaps qa's qubits with result's. But the CIRCUIT still has the gates operating on the physical qubits. The physical layout is: qa_orig at [0..w-1], ret at [w..2w-1]. The product is computed into ret. After imul, the Python object qa now refers to ret's physical qubits [w..2w-1]. But from the QASM perspective, we need to measure qubit positions [w..2w-1] to get the result.
- So `_verify_toffoli_mul_cq` extracts from result_start = width (after the original self qubits).
