Fix BUG-06 (MSB comparison ancilla leak) and BUG-08 (QFT division/modulo failures).

Purpose: The division loop calls `remainder >= trial_value` at each bit position. The `>=` operator delegates to `~(self < other)`, which creates widened temporary qint objects (comp_width = max_bits + 1) for the MSB sign-bit subtraction. These temporaries are Python objects subject to garbage collection, but their qubit allocations can conflict across loop iterations, corrupting comparison results for values where the MSB is set (a >= 2^(w-1)). Per CONTEXT.md, fix BUG-06 first and then verify if BUG-08 resolves.

**Diagnosis approach:**

The 9 known-failing cases in `KNOWN_DIV_MSB_LEAK` (test_div.py) all share a pattern: `a >= 2^(w-1)` with small divisors. The `remainder >= trial_value` comparison in the division loop:

1. `__ge__` calls `~(self < other)`
2. `__lt__` for `qint < int` creates a temporary `temp = qint(other, width=self.bits)` and recurses to `qint < qint`
3. `qint < qint` creates widened temps: `temp_self = qint(0, width=comp_width)`, `temp_other = qint(0, width=comp_width)` where `comp_width = max_bits + 1`
4. Bit-by-bit CNOT copy into widened temps
5. `temp_self -= temp_other`
6. MSB extraction: `msb = temp_self[comp_width - 1]`, `result ^= msb`

**Root cause investigation:**

The widened temporaries `temp_self` and `temp_other` in `__lt__` (qint_comparison.pxi) are local Python variables that go out of scope when `__lt__` returns. However, their qubits were allocated by the qubit allocator and are NOT freed until the `qint.__del__` destructor runs (which depends on Python garbage collection).

In the division loop, each iteration's `can_subtract = remainder >= trial_value` creates new widened temps. If GC has not run, the old temps' qubits are still "allocated" in the allocator, so new temps get fresh qubits. But the old temps' GATES (subtraction, CNOT copies) are still in the circuit. When the old temps are eventually GC'd, their `__del__` may inject uncomputation gates at unpredictable times.

**Fix strategy:**

Option A (preferred): Add explicit garbage collection (`gc.collect()`) at the end of each loop iteration in `__floordiv__` and `__mod__`. This forces cleanup of comparison temporaries before the next iteration.

Option B: Modify `__lt__`/`__gt__` to explicitly uncompute and free widened temporaries within the comparison function itself, before returning only the result bit.

**Try Option A first** — add `import gc` at the top of the method and insert `gc.collect()` after the `with can_subtract:` block in:
- `__floordiv__` (qint_division.pxi, line ~71 area, inside the `for bit_pos` loop)
- `__mod__` (qint_division.pxi, line ~192 area, inside the `for bit_pos` loop)
- `__divmod__` (qint_division.pxi, line ~296 area, inside the `for bit_pos` loop)

If Option A does not fix the issue (check against the 9 known-failing cases), proceed to Option B. For Option B, modify `__lt__` in qint_comparison.pxi to add explicit uncomputation of the widened temporaries after extracting the MSB result:
```python
# After: result ^= msb
# Restore: temp_self += temp_other (undo the subtraction)
temp_self += temp_other
# Uncopy: reverse the bit-by-bit CNOT copies
for i_bit in range(operand_bits - 1, -1, -1):
    qubit_array[0] = temp_other.qubits[64 - comp_width + i_bit]
    qubit_array[1] = other.qubits[64 - operand_bits + i_bit]
    # ... reverse CNOT
for i_bit in range(self.bits - 1, -1, -1):
    qubit_array[0] = temp_self.qubits[64 - comp_width + i_bit]
    qubit_array[1] = self.qubits[64 - operand_bits + i_bit]
    # ... reverse CNOT
```

Also apply the same fix to `__gt__` which has an identical widened-temp pattern.
