**Diagnosis approach:** The cQQ_add function implements Beauregard's controlled addition via CCP decomposition. The CCP(theta) gate on target t, controlled by c1 (external control) and c2 (b-register), decomposes as:

```
CCP(theta)_{c1,c2,t} = CP(theta/2)_{c1,t} . CNOT_{c1,c2} . CP(-theta/2)_{c2,t} . CNOT_{c1,c2} . CP(theta/2)_{c2,t}
```

The current code has three blocks:
- **Block 1** (lines ~434-443): Unconditional half-rotations. Accumulates `value += 2*PI / 2^(i+1) / 2` for each control bit contributing to target qubit. The accumulated value is applied as `cp(g, target_q, control, value)`.
- **Block 2** (lines ~447-462): CNOT + negative half-rotations + CNOT. Individual `-value` rotations per source bit.
- **Block 3** (lines ~466-476): Positive half-rotations from b-register. Individual `+value` rotations per source bit.

**What to verify:**
1. The rotation angle for each target qubit from each source bit should be `2*PI / 2^(j-k+1)` where j is the target Fourier mode and k is the source bit position.
2. Block 1's accumulation formula should sum the half-angles: for target j, sum over k from 0 to j of `PI / 2^(j-k+1)`.
3. Block 2's negative rotations should use individual angles `-PI / 2^(j-k+1)` (not accumulated).
4. Block 3's positive rotations should use individual angles `+PI / 2^(j-k+1)` (not accumulated).

**Compare with QQ_add:** The unconditioned QQ_add (which works correctly) uses full rotation angles. cQQ_add should produce the SAME result when control=1. Write a small test that generates both circuits and compares gate-by-gate (or compare expected output for a known input like width=2, a=1, b=1, ctrl=1 should give a=2).

**Fix the rotation formulas** in the dynamic generator and in BOTH Python generation scripts (`generate_seq_1_4.py` and `generate_seq_5_8.py`) to ensure consistency.

**Build after fixing:** `python3 setup.py build_ext --inplace`
