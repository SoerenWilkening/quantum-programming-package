<objective>
Fix controlled multiplication scope uncomputation corruption (BUG-07/BUG-COND-MUL-01) by preventing multiplication results from being registered in the scope frame.

Purpose: When `result *= 2` runs inside a `with cond:` block, `__mul__` creates a new qint that gets registered in the scope frame. When `__exit__` runs, it uncomputes ALL scope-registered qints including the multiplication result, reversing all multiplication gates and producing incorrect output. The fix: temporarily set `current_scope_depth` to 0 during result qint creation in `__mul__`/`__rmul__`, preventing scope registration. This approach is proven correct in Phase 69-02 tests.
Output: Working controlled multiplication, regression tests, removed xfail markers.
</objective>

  <files>src/quantum_language/qint_arithmetic.pxi</files>
  <action>
1. Read `src/quantum_language/qint_arithmetic.pxi` and locate the `__mul__` method.

2. In `__mul__`, immediately before the line that creates the result qint (`result = qint(width=...)`), add scope depth bypass:
   ```python
   # BUG-COND-MUL-01 fix: prevent result registration in scope frame.
   # Multiplication results should never be auto-uncomputed by scope exit.
   # The user explicitly assigns them; cleanup is handled by GC, not scope.
   _saved_scope = current_scope_depth.get()
   current_scope_depth.set(0)
   ```

3. Immediately after the result qint is created (before `multiplication_inplace` or any other operation on it), restore scope depth:
   ```python
   current_scope_depth.set(_saved_scope)
   ```

4. Apply the same pattern to `__rmul__` if it also creates a result qint (it may delegate to `__mul__`; check the code).

5. **Critical: Do NOT modify `__init__`, `__exit__`, or the scope registration logic broadly.** Only modify the multiplication code path. This is per user decision: "targeted patch, minimal blast radius."

6. **Do NOT modify `__imul__` for this fix.** `__imul__` calls `self * other` which goes through `__mul__`. The fix in `__mul__` automatically fixes the in-place case too.

7. Rebuild: `pip install -e .`
