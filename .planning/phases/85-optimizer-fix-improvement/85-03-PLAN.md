**Optimization 1: Eliminate per-gate malloc in inject_remapped_gates**

In `_core.pyx`, replace the per-gate `malloc(sizeof(gate_t))` + `memset` with stack allocation for the common case (NumControls <= 2). Use a stack-allocated `gate_t` variable:

```cython
def inject_remapped_gates(list gates, dict qubit_map):
    cdef circuit_t *circ
    cdef gate_t g  # Stack-allocated, reused per gate
    cdef int i
    _validate_circuit()
    circ = <circuit_t*>_circuit
    for gate_dict in gates:
        memset(&g, 0, sizeof(gate_t))
        g.Gate = <Standardgate_t>gate_dict['type']
        g.Target = <qubit_t>qubit_map[gate_dict['target']]
        g.GateValue = gate_dict['angle']
        g.NumControls = <unsigned int>gate_dict['num_controls']
        controls = gate_dict['controls']
        if g.NumControls <= 2:
            for i in range(<int>g.NumControls):
                g.Control[i] = <qubit_t>qubit_map[controls[i]]
        else:
            g.large_control = <qubit_t*>malloc(g.NumControls * sizeof(qubit_t))
            if g.large_control == NULL:
                raise MemoryError("Failed to allocate large_control array")
            for i in range(<int>g.NumControls):
                g.large_control[i] = <qubit_t>qubit_map[controls[i]]
            g.Control[0] = g.large_control[0]
            g.Control[1] = g.large_control[1]
        add_gate(circ, &g)
```

Key change: `gate_t g` on the stack instead of `malloc(sizeof(gate_t))`. The `add_gate` function copies gate data via `memcpy` in `append_gate`, so the stack variable is safe. For MCX gates (NumControls > 2), `large_control` is still heap-allocated because `add_gate` stores the pointer.

IMPORTANT: Verify that `add_gate` copies the gate (via memcpy in append_gate). If it stores a pointer, stack allocation would be unsafe. Read optimizer.c `append_gate` to confirm it does `memcpy(&circ->sequence[layer][pos], g, sizeof(gate_t))` -- this copies the full struct, so stack allocation is safe. The `large_control` pointer is copied as part of the struct, which is correct since the heap-allocated `large_control` array persists.
