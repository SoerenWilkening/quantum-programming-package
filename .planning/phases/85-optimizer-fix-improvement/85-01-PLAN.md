**Part A: Fix the bug in optimizer.c**

In `smallest_layer_below_comp` (line 32), change `++i` to `--i`:

Before: `for (int i = last_index; i > 0; ++i)`
After:  `for (int i = last_index; i > 0; --i)`

Also fix the boundary check: change `if (last_index < 0)` to `if (last_index <= 0)` since `last_index` is `used_occupation_indices_per_qubit[qubit]` which is unsigned (`num_t` = `unsigned int`) cast to int, and an index of 0 means no occupied layers.

**Part B: Create golden-master test infrastructure**

Create `tests/python/test_optimizer_golden_master.py` with:

1. A `capture_circuit_snapshot(name, setup_func)` helper that:
   - Calls `ql.circuit()` to create fresh circuit
   - Calls `setup_func()` to build the circuit
   - Uses `extract_gate_range(0, get_current_layer())` to extract all gates
   - Returns a dict with `circuit_name`, `total_gates`, `gates` (list of gate dicts with type, target, controls, angle, and layer info from the circuit)

2. A `save_golden_master(name, snapshot)` function that writes JSON to `tests/golden_masters/{name}.json`

3. A `load_golden_master(name)` function that reads JSON from `tests/golden_masters/{name}.json`

4. A `compare_snapshots(expected, actual)` function that compares two snapshots gate-by-gate and returns a diff if they don't match

5. A curated set of 15-20 representative circuit functions covering:
   - Addition: 2-bit, 4-bit, 8-bit (toffoli arithmetic mode)
   - Subtraction: 4-bit
   - Multiplication: 2-bit, 4-bit
   - Comparison: 4-bit (lt, eq)
   - Bitwise: 4-bit XOR, AND
   - QFT addition: 4-bit (QFT arithmetic mode)
   - Grover search: simple 3-qubit predicate (if grover available)
   - Compiled function: simple `@ql.compile` add_one on 4-bit
   - Controlled operation: controlled addition 4-bit
   - Mixed: sequence of add + sub on same register

6. A `test_generate_golden_masters` test (marked with `@pytest.mark.golden_master_gen`) that generates all golden-master JSON files. This should be run manually to regenerate snapshots, NOT in CI.

7. A `test_verify_golden_masters` test that loads each golden-master file and compares against a freshly generated circuit. This runs in normal pytest and fails if any circuit output differs from the committed snapshot.

Important: For gate extraction, use `extract_gate_range` from `quantum_language._core`. The gate dicts already contain `type`, `target`, `controls`, `angle`, `num_controls`. To capture layer information, iterate through circuit layers using the circuit stats or record gates with their layer index.

Note: The circuit may use different arithmetic modes. Default to toffoli mode (`ql.set_arithmetic_mode('toffoli')`) for deterministic output. Include a few QFT-mode circuits as well.

Store golden-master JSON files in `tests/golden_masters/` directory (create it).
