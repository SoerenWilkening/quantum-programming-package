<task type="auto">
  <name>Task 1: Add predicate-to-oracle synthesis to oracle.py</name>
  <files>src/quantum_language/oracle.py</files>
  <action>
Add two new functions to oracle.py:

1. `_lambda_cache_key(predicate, register_widths)` -- Build cache key for lambda oracles including closure variable values.
   - Compute source hash via existing `_compute_source_hash(predicate)`
   - Extract closure variables: `predicate.__closure__` cell contents, filtering to (int, float, str, bool) types only
   - Include `arithmetic_mode` (1 if `option("fault_tolerant")` else 0) and `tuple(register_widths)` in key
   - Return tuple: `(src_hash, closure_values, arithmetic_mode, tuple(register_widths))`

2. `_predicate_to_oracle(predicate, register_widths)` -- Convert Python callable predicate into GroverOracle.
   - Import `inspect`, `math` at top; import `compile as ql_compile` from `.compile`, `qint` from `.qint`
   - Use `inspect.signature(predicate).parameters` to determine parameter count
   - Create a wrapper function that:
     a. Accepts qint arguments matching the predicate's parameter count
     b. Calls `predicate(*qint_args)` to trace gates (returns qbool via operator overloading)
     c. Validates that result is a qint/qbool (raise TypeError if predicate returns Python bool/None)
     d. Applies phase marking: `with result: args[0].phase += math.pi`
   - Wrap the wrapper with `ql_compile` decorator, then with `GroverOracle(compiled, validate=False)`
   - Use `_lambda_cache_key` to check a module-level `_predicate_oracle_cache` dict before building
   - On cache miss, build the traced oracle and cache it
   - Return the GroverOracle instance

Important implementation notes:
- The tracing approach means the predicate IS called with real qint objects -- existing qint operators (`__gt__`, `__lt__`, `__eq__`, `__and__`, `__or__`, `__invert__`) capture gates automatically
- Compound predicates like `(x > 10) & (x < 50)` work because each comparison returns qbool, and `&` is already implemented in qint_bitwise.pxi
- `validate=False` is required because phase marking targets the comparison ancilla qubit, not search register qubits (same pattern as Phase 79)
- DO NOT use AST parsing -- tracing reuses all existing operator infrastructure
- Use `with result: args[0].phase += math.pi` for phase marking (NOT `with result: pass` which is a no-op after compile optimization, per Phase 79-02 decision)

Export `_predicate_to_oracle` and `_lambda_cache_key` from oracle.py (they are private but needed by grover.py).
