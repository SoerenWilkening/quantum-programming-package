  <files>src/quantum_language/qint_comparison.pxi</files>
  <action>
Fix the hardcoded qubit index 63 for MSB access in the inequality comparison operators. There are exactly 2 lines to change in `src/quantum_language/qint_comparison.pxi`:

1. **Line 301** (in `__lt__`, qint-qint path):
   Change: `msb = temp_self[63]`
   To: `msb = temp_self[comp_width - 1]`

   Context: `temp_self` is created with `width=comp_width` where `comp_width = max(self.bits, other.bits) + 1`. The MSB is at index `comp_width - 1` (0-indexed from LSB). `__getitem__` bounds-checks against `.bits`, so index 63 fails for any `comp_width < 64`.

2. **Line 421** (in `__gt__`, qint-qint path):
   Change: `msb = temp_other[63]`
   To: `msb = temp_other[comp_width - 1]`

   Context: Same logic. `temp_other` has `width=comp_width`. The MSB (sign bit of the widened subtraction) is at `comp_width - 1`.

**Why only these 2 lines:** The other `[63]` accesses in qint_comparison.pxi (lines 129 and 141) access `.qubits[63]` directly on qbool objects (width=1, qubit stored right-aligned at internal array position 63). These access the raw C array, NOT through `__getitem__`, and are correct.

**Why `comp_width - 1` is correct:** The temporary qints are `width=comp_width`. `__getitem__` uses 0-indexed qubit positions where 0=LSB and `width-1`=MSB. The MSB of the widened subtraction result is the sign bit, at position `comp_width - 1`.

**Cascading fixes:** `__le__` delegates to `~(self > other)` and `__ge__` delegates to `~(self < other)`. Fixing `__lt__` and `__gt__` automatically fixes all four inequality operators.
