**2. Exhaustive CQ addition tests (widths 2-4, statevector):**
Generate test cases that cover ALL (a, b) combinations for each modulus:
- Width 2: N in {2, 3} (2-bit values, 0 to N-1)
- Width 3: N in {3, 5, 7} (3-bit values, 0 to N-1)
- Width 4: N in {5, 7, 9, 11, 13, 15} (4-bit values, 0 to N-1)

For each (N, a, b) where a in [0, N-1] and b in [0, N-1]:
  expected = (a + b) % N
  Simulate and assert actual == expected

Use pytest.mark.parametrize with a generator function. Each test creates a circuit, performs the operation, and verifies the result.

IMPORTANT: Max 17 qubits for statevector, 4 threads max for AerSimulator.

**3. Exhaustive CQ subtraction tests (widths 2-4, statevector):**
Same structure as addition but with subtraction:
  expected = (a - b) % N

**4. Exhaustive CQ multiplication tests (widths 2-4, statevector):**
For each (N, a, c) where a in [0, N-1] and c is classical int in [0, N-1]:
  expected = (a * c) % N
Test multiply by 0 returns 0.

**5. QQ addition tests (widths 2-3, statevector):**
For each (N, a, b) where both a and b are qint_mod:
  expected = (a + b) % N
Width 2-3 only (QQ operations use more qubits). Verify same-N requirement:
  qint_mod(a, N=5) + qint_mod(b, N=7) raises ValueError.

**6. QQ subtraction tests (widths 2-3, statevector):**
For each (N, a, b) where both are qint_mod:
  expected = (a - b) % N

**7. QQ multiplication tests (widths 2-3, statevector):**
For each (N, a, b) where both are qint_mod:
  expected = (a * b) % N
Width 2-3 only due to qubit budget (QQ mul uses 2n-bit product register + ancillae).

**8. Controlled modular operation tests:**
Test that `with ctrl_qubit: a += b` (controlled modular add) works:
- When control = |1>: result is (a + b) mod N
- When control = |0>: result is a (unchanged)
Test both CQ and QQ variants. Use representative (not exhaustive) inputs at width 3.

**9. Negation tests:**
For each N in {3, 5, 7} and a in [0, N-1]:
  r = -qint_mod(a, N=N)
  expected = (N - a) % N
  Assert actual == expected

**10. In-place operator tests:**
Test +=, -=, *= for representative inputs. Verify the variable is updated correctly.

**11. Type infection tests:**
- qint_mod(3, N=5) + qint(2, width=3) returns qint_mod with modulus 5
- Verify result type is qint_mod and result.modulus == 5

**12. Validation tests:**
- qint_mod(0, N=0) raises ValueError
- qint_mod(0, N=1) raises ValueError
- qint_mod(a, N=5) + qint_mod(b, N=7) raises ValueError (mismatched moduli)
- qint_mod(7, N=5) constructs with value 2 (auto-reduce)

**Test helper updates:**
Keep the existing `_simulate_and_extract` helper but update the `_run_modular_op` helper to support QQ operations. Add a `_run_modular_qq_op` for operations between two qint_mod values.

**Qubit budget awareness:**
- Statevector: max 17 qubits. For CQ add at width 4: value(4) + result(4) + ancillae(~6) = ~14 qubits. OK.
- For CQ mul at width 4: value(4) + result(4) + ancillae per iteration(~6) = ~14 peak. OK.
- For QQ add at width 3: value(3) + other(3) + result(3) + ancillae(~5) = ~14. OK.
- For QQ mul at width 3: a(3) + b(3) + result(3) + product(6) + ancillae(~6) = ~21. Use MPS.
- For QQ mul at width 2: a(2) + b(2) + result(2) + product(4) + ancillae(~4) = ~14. Statevector OK.

Adjust width ranges per operation type to stay within qubit budget.
