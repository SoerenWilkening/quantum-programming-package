Rewrite `toffoli_mod_add_cq` in ToffoliModReduce.c to use the Beauregard 8-step modular addition sequence with clean ancilla uncomputation. The current implementation (add + reduce) leaks 1 persistent ancilla per call. Replace it entirely.

**Beauregard CQ modular addition (value += a mod N):**
Both operands in [0, N-1]. Allocate 1 high_bit ancilla + 1 cmp_anc ancilla. Build (n+1)-bit wide register = [value_qubits[0..n-1], high_bit].

Step 1: mod_cq_add(wide_reg, n+1, a)        -- value += a
Step 2: mod_cq_add(wide_reg, n+1, -N)       -- value -= N
Step 3: mod_emit_cx(cmp_anc, wide_reg[n])   -- copy sign to cmp_anc
Step 4: mod_ccq_add(wide_reg, n+1, N, cmp_anc) -- if cmp_anc=1: value += N
Step 5: mod_cq_add(wide_reg, n+1, -a)       -- value -= a (undo step 1)
Step 6: mod_emit_x(cmp_anc)                 -- flip cmp_anc
Step 7: mod_emit_cx(cmp_anc, wide_reg[n])   -- reset cmp_anc using sign bit
Step 8: mod_cq_add(wide_reg, n+1, a)        -- value += a (redo step 1)

After step 8: value = (orig + a) mod N, cmp_anc = 0, high_bit = 0. Free both ancillae.

Edge cases:
- If a == 0: return immediately (no-op).
- Classically reduce: a = addend % modulus. If a < 0: a += modulus.
- For 1-bit width (n=1): special case using mod_cq_add with bits=1.

**Controlled CQ modular addition (toffoli_cmod_add_cq):**
Same 8 steps but ALL additions are controlled by ext_ctrl. Steps 1,2,5,8 use mod_ccq_add(reg, n+1, value, ext_ctrl). Step 4 becomes doubly-controlled (both cmp_anc AND ext_ctrl): use AND-ancilla pattern (CCX(and_anc, cmp_anc, ext_ctrl); mod_ccq_add(..., and_anc); CCX(and_anc, cmp_anc, ext_ctrl)). Steps 3,7 become controlled CNOT (= CCX with ext_ctrl). Step 6 becomes controlled X (= CX with ext_ctrl).

**Modular CQ subtraction (toffoli_mod_sub_cq):**
Wrapper: (value - b) mod N = (value + (N - b)) mod N.
Compute complement = (modulus - (subtrahend % modulus)) % modulus. Call toffoli_mod_add_cq with complement.

**Controlled modular CQ subtraction (toffoli_cmod_sub_cq):**
Same wrapper pattern but calls toffoli_cmod_add_cq.

Add all new function declarations to toffoli_arithmetic_ops.h:
- toffoli_mod_sub_cq
- toffoli_cmod_sub_cq

- toffoli_cmod_add_qq

- toffoli_cmod_sub_qq

Keep the existing `toffoli_mod_reduce` and `toffoli_cmod_reduce` functions (they may be useful for other purposes) but the mod_add_cq/cmod_add_cq implementations must be REPLACED with the Beauregard sequence.

IMPORTANT: Do NOT use hot_path dispatch. Call toffoli_CQ_add / toffoli_cCQ_add directly (forces RCA/CDKM, which is required because CLA subtraction cannot be inverted).

**QQ modular addition (toffoli_mod_add_qq): value += other_quantum mod N**
Same Beauregard 8-step structure as CQ, but using QQ adders instead of CQ adders for steps involving the quantum operand (other). The key insight from the RESEARCH.md: CDKM QQ add preserves the source register b (only modifies the target a). So `other` can be used directly in steps 5 and 8 without needing a copy.

Allocate high_bit + cmp_anc. Build wide register = [value_qubits[0..n-1], high_bit].

Create a new helper `mod_qq_add` (analogous to `mod_cq_add`) that calls `toffoli_QQ_add(bits)` with the correct qubit layout: [temp_a[0..bits-1], other_b[0..bits-1], carry]. For the (n+1)-bit operations, extend other with a zero ancilla for bit n.

Step 1: mod_qq_add(wide_reg, n+1, other_qa, n)  -- value += other
Step 2: mod_cq_add(wide_reg, n+1, -N)            -- value -= N (N is classical)
Step 3: mod_emit_cx(cmp_anc, wide_reg[n])         -- copy sign
Step 4: mod_ccq_add(wide_reg, n+1, N, cmp_anc)   -- controlled add N (N is classical, controlled by cmp_anc)
Step 5: mod_qq_add_inv(wide_reg, n+1, other_qa, n) -- value -= other (inverse QQ add)
Step 6: mod_emit_x(cmp_anc)
Step 7: mod_emit_cx(cmp_anc, wide_reg[n])
Step 8: mod_qq_add(wide_reg, n+1, other_qa, n)    -- value += other

For step 5 (inverse QQ add): use `run_instruction(seq, tqa, 1, circ)` with invert=1 to run the CDKM QQ adder in reverse (= subtraction).

For the (n+1)-bit QQ operations: the other register only has n bits. Pad with a zero ancilla at position n. Allocate 1 extra qubit (other_high), keep it at |0>, place at other_qa[n]. After the operation, free it. This ensures the (n+1)-bit QQ add only adds the n-bit value.

**Helper functions needed:**
- `mod_qq_add(circ, target_reg, target_bits, source_reg, source_bits)`: Performs target += source using toffoli_QQ_add. Allocates carry ancilla. For target_bits > source_bits, pads source with zero ancillae.
- `mod_qq_add_inv(circ, target_reg, target_bits, source_reg, source_bits)`: Same but inverted (target -= source).
- `mod_cqq_add(circ, target_reg, target_bits, source_reg, source_bits, ctrl)`: Controlled QQ add using toffoli_cQQ_add.
- `mod_cqq_add_inv(circ, target_reg, target_bits, source_reg, source_bits, ctrl)`: Controlled inverse QQ add.

**Controlled QQ modular addition (toffoli_cmod_add_qq):**
All QQ additions in the sequence become controlled by ext_ctrl. Steps involving N (classical) use mod_ccq_add with ext_ctrl. Step 4 becomes doubly-controlled (cmp_anc AND ext_ctrl) -- use AND-ancilla pattern.

**QQ modular subtraction (toffoli_mod_sub_qq and toffoli_cmod_sub_qq):**
Cannot use the N-complement trick (other is quantum). Instead: compute (N - other) into a temp register, do modular QQ add of temp, uncompute temp.

Alternative (simpler): Use the Beauregard sequence directly but with inversion. The subtraction (value -= other mod N) is equivalent to the INVERSE of (value += other mod N). Since the Beauregard sequence is reversible, run all 8 steps in reverse order with each addition/subtraction inverted.

Implement as: run_instruction on each step's sequence with invert flag toggled.

OR even simpler: implement `toffoli_mod_sub_qq` as a separate 8-step sequence where the addition/subtraction roles are swapped. Step 1: value -= other. Step 2: value += N (instead of -N). Step 3-7: same sign logic (adjusted). Step 8: value -= other.

Recommended approach: Compute `temp = N - other` (classical N minus quantum other) using CQ_add of N into a temp register then QQ_sub of other. Then call toffoli_mod_add_qq(value, temp, N). Then uncompute temp. This is clean and reuses existing primitives.

Actually the simplest correct approach: negate other into temp (temp = N - other), mod_add_qq(value, temp, N), then uncompute temp. Negation: temp = 0, temp += N (CQ), temp -= other (QQ inverse). Uncompute: temp += other (QQ), temp -= N (CQ), now temp = 0.

Declare all 4 QQ functions in toffoli_arithmetic_ops.h.
