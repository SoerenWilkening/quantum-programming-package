**Step 2: Create `oracle.py`**

Create `src/quantum_language/oracle.py` with the following structure:

1. **Imports:** `hashlib`, `inspect`, `collections`, `functools` from stdlib. From project: `CompiledFunc` from `.compile`, `compile` function from `.compile`, `circuit_stats` from `._core`, `_allocate_qubit`, `_deallocate_qubits` from `._core`, `option` from `._core`, `extract_gate_range`, `get_current_layer`, `inject_remapped_gates` from `._core`, `emit_x`, `emit_h` from `._gates`, gate type constants `_Z`, `_SELF_ADJOINT`, `_ROTATION_GATES` from `.compile`.

2. **`_compute_source_hash(func)`**: Takes the original function (not CompiledFunc wrapper). Uses `inspect.getsource(func)` with `hashlib.sha256`. Falls back to `str(id(func.__code__))` on `OSError`/`TypeError`. Returns 16-char hex string.

3. **`_oracle_cache_key(func, register_width)`**: Builds cache key tuple `(source_hash, arithmetic_mode_int, register_width)`. Gets arithmetic_mode from `option('fault_tolerant')` (1 if True, 0 if False).

4. **`_validate_ancilla_delta(pre_stats, post_stats)`**: Compares `current_in_use` values. Raises `ValueError` with message: `f"Oracle ancilla delta is {delta} (must be 0). Ensure all temporary qubits are uncomputed."` if delta != 0.

5. **`_validate_compute_phase_uncompute(gates, param_qubit_count)`**: Post-hoc analysis of captured gate sequence. Strategy:
   - Identify all phase gates: gates where `type` is `_Z` (type value 2) AND the target qubit index is within param range (0..param_qubit_count-1). Also count CZ gates (type Z with num_controls >= 1) targeting param qubits.
   - If no phase gates found, raise `ValueError("Oracle has no phase-marking gates on search register qubits.")`.
   - Find the index of the first phase gate. Split gates into `before_phase` (gates before first phase gate) and `after_phase` (gates after last phase gate).
   - Verify `after_phase` is approximately the adjoint of `before_phase`: reversed order, and for each pair, check using `_gates_cancel` from compile.py. If verification fails, raise `ValueError("Oracle does not follow compute-phase-uncompute pattern.")`.
   - NOTE: This validation is best-effort. Complex oracles (e.g., using QFT arithmetic internally) may have Z-type gates in the compute phase that target ancilla qubits -- those should NOT be treated as the phase boundary. Only Z gates targeting param qubits (virtual indices 0..width-1) count.

6. **`_wrap_bitflip_oracle(oracle_callable, search_register)`**: Phase kickback wrapping:
   - Record `circuit_stats()['current_in_use']` as pre_ancilla
   - Allocate ancilla: `ancilla_q = _allocate_qubit()`
   - Save and clear controlled context (X and H on ancilla must be unconditional even if called inside `with qbool:`)
   - `emit_x(ancilla_q)` then `emit_h(ancilla_q)` -- creates |-> state
   - Restore controlled context
   - Call `oracle_callable(search_register)` -- user oracle runs in original context
   - Save and clear controlled context again
   - `emit_h(ancilla_q)` then `emit_x(ancilla_q)` -- undo |-> preparation
   - Restore controlled context
   - `_deallocate_qubits(ancilla_q, 1)` -- ancilla back to |0>, deallocate
   - After wrapping, check the captured gates to verify at least one gate targeted the ancilla qubit. If no gate targets it, raise `ValueError("bit_flip=True but oracle did not interact with ancilla target qubit. Mismatch.")`

7. **`GroverOracle` class**:
   - `__init__(self, compiled_func, bit_flip=False, validate=True)`:
     - Store `compiled_func` (a CompiledFunc instance), `bit_flip`, `validate`
     - Store `_original_func` = compiled_func._func (for source hashing)
     - `_cache` = `collections.OrderedDict()` with max 128 entries
     - `functools.update_wrapper(self, compiled_func._func)`
   - `__call__(self, *args, **kwargs)`:
     - Extract search register (first quantum arg) to get width
     - Build oracle cache key using `_oracle_cache_key(self._original_func, width)`
     - If cache hit: replay cached oracle block (use `inject_remapped_gates` with the cached block's gates and a virtual-to-real mapping)
     - If cache miss:
       a. Record `pre_stats = circuit_stats()`
       b. If `bit_flip=True`: call `_wrap_bitflip_oracle` which internally calls the compiled func
       c. If `bit_flip=False`: call `self._compiled_func(*args, **kwargs)` directly
       d. Record `post_stats = circuit_stats()`
       e. If `validate=True`:
          - Call `_validate_ancilla_delta(pre_stats, post_stats)`
          - Optionally validate compute-phase-uncompute (extract gate range and analyze)
       f. Cache the result
     - Return result from compiled func call
   - `__repr__`: `f"<GroverOracle {self._original_func.__name__}>"`

8. **`grover_oracle` decorator function**: Supports three forms:
   - `@ql.grover_oracle` (bare, wrapping a CompiledFunc or plain function)
   - `@ql.grover_oracle()` (empty parens)
   - `@ql.grover_oracle(bit_flip=True, validate=False)` (with kwargs)

   Implementation pattern (per CONTEXT.md locked decision):
   ```python
   def grover_oracle(func=None, *, bit_flip=False, validate=True):
       def decorator(fn):
           if not isinstance(fn, CompiledFunc):
               fn = compile(fn)
           return GroverOracle(fn, bit_flip=bit_flip, validate=validate)
       if func is not None:
           if isinstance(func, CompiledFunc):
               return GroverOracle(func, bit_flip=bit_flip, validate=validate)
           return GroverOracle(compile(func), bit_flip=bit_flip, validate=validate)
       return decorator
   ```

**Important design decisions:**
- Validation happens at circuit generation time (first call), not decoration time -- per CONTEXT.md Claude's discretion
- `validate=False` bypasses ALL checks (both ordering and ancilla delta) -- per research recommendation
- Cache is global/persistent within Python process -- per CONTEXT.md locked decision
- The GroverOracle delegates gate capture to the inner CompiledFunc; it adds validation and wrapping on top
- For the controlled context save/restore in bit-flip wrapping, use `_get_controlled()`, `_set_controlled()`, `_get_control_bool()`, `_set_control_bool()`, `_get_list_of_controls()`, `_set_list_of_controls()` from `._core` (same pattern as `CompiledFunc._capture_and_cache_both`)
