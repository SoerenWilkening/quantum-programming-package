**Group 5: Oracle Caching (ORCL-05)**

- `test_cache_includes_arithmetic_mode`: Call oracle with `fault_tolerant=False` then `fault_tolerant=True` -- produces different cache entries. Use `ql.option('fault_tolerant', True)` and `ql.option('fault_tolerant', False)` to switch modes. Verify both calls succeed.

**Qiskit Verification Pattern (from test_branch_superposition.py):**
```python
from qiskit import transpile
from qiskit_aer import AerSimulator
from quantum_language import to_openqasm

def simulate(circ, shots=8192):
    qasm_str = to_openqasm(circ)
    from qiskit import QuantumCircuit
    qc = QuantumCircuit.from_qasm_str(qasm_str)
    qc.measure_all()
    simulator = AerSimulator()
    qc_t = transpile(qc, simulator)
    result = simulator.run(qc_t, shots=shots).result()
    return result.get_counts()
```

**Important test conventions:**
- Every test starts with `ql.circuit()` to get a fresh circuit
- Remember Qiskit little-endian bit ordering: qubit[0] is RIGHTMOST bit in bitstrings
- Use `@pytest.fixture` for common setup if beneficial
- Import: `import quantum_language as ql`, `from quantum_language.oracle import GroverOracle`
