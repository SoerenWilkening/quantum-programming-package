<objective>
Optimize qarray element-wise operations with classical (int/list) operands to call CQ_* C backend functions directly, eliminating temporary qint allocations per element.

Purpose: Reduce circuit-generation overhead for array-scalar operations. Currently each element creates a temporary qint for the classical value before calling the in-place operator. This optimization bypasses that allocation and calls the C backend directly.

Output: Modified `qarray.pyx` with optimized `_inplace_binary_op` for classical operands and optimized out-of-place paths in `_elementwise_binary_op`.

  **Step 3: Add optimized `_inplace_classical_mul` method.**

  For multiplication, the layout is: [ret_qubits, self_qubits, ancilla]. Since in-place mul in qint creates a new result and swaps, the qarray in-place mul must also allocate a result qint per element. But the optimization is avoiding the intermediate `qint(other, width=self._width)` temporary for the classical value. Follow the same pattern as qint.multiplication_inplace but call CQ_mul directly:

  ```python
  cdef _inplace_classical_mul(self, long long value):
      """In-place multiply each element by classical value using CQ_mul directly."""
      cdef sequence_t *seq
      cdef unsigned int[:] arr
      cdef int result_bits, self_offset, ret_offset, start
      cdef circuit_t *_circuit = <circuit_t*><unsigned long long>_get_circuit()

      if _get_controlled():
          return self._inplace_binary_op(value, "__imul__")

      for i in range(len(self._elements)):
          elem = <qint>self._elements[i]
          result_bits = elem.bits
          # Allocate result qint (accumulator)
          ret = qint(width=result_bits)
          self_offset = 64 - elem.bits
          ret_offset = 64 - result_bits
          # Layout: [ret, self, ancilla]
          _qa_qubit_array[:result_bits] = ret.qubits[ret_offset:64]
          _qa_qubit_array[result_bits: result_bits + elem.bits] = elem.qubits[self_offset:64]
          start = result_bits + elem.bits
          _qa_qubit_array[start: start + NUMANCILLY] = _get_ancilla()
          seq = CQ_mul(result_bits, value)
          if seq == NULL:
              raise RuntimeError(f"Multiplication circuit generation failed for width {result_bits}")
          arr = _qa_qubit_array
          run_instruction(seq, &arr[0], False, _circuit)
          # Swap element to result (same pattern as qint.__imul__)
          self._elements[i] = ret
      return self
  ```

  **Step 6: Optimize `_elementwise_binary_op` for scalar int broadcast.**

  In the `_elementwise_binary_op` method, when `type(other) == int`, the lambda dispatch `op_func(elem, other)` calls qint's `__add__` which internally does: `a = qint(value=self.value, width=result_width); a += other` which already calls CQ_add. The out-of-place path is harder to optimize without duplicating significant qint logic (dependency tracking, layer tracking, value tracking for simulation).

  **Decision: Only optimize in-place operations.** Out-of-place operations (`a + 3`) inherently need a new qint object for the result, and qint's __add__ already calls CQ_add internally. The overhead is the lambda dispatch (negligible) and the qint copy (required for out-of-place semantics). No meaningful optimization available without breaking encapsulation.
