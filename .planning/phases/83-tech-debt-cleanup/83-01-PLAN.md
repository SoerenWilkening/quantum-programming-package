    Add a `sync_and_stage()` function that:
    1. Iterates over all .pyx files in SRC_DIR that have include directives (same logic as preprocess_all)
    2. For each, reads the existing preprocessed file content (if it exists)
    3. Calls process_includes() to regenerate the preprocessed output
    4. Compares old vs new content
    5. If different: prints "Drift detected: {filename} -- regenerated and staging", then runs `subprocess.run(["git", "add", str(output)], check=True)` to auto-stage
    6. If same: no action, exits cleanly
    7. Returns 0 always (auto-fix hook should not block commits after fixing)

    Add `import subprocess` at the top of the file.

    Update the `if __name__ == "__main__"` block to handle `--sync-and-stage`:
    ```python
    if "--sync-and-stage" in sys.argv:
        sys.exit(sync_and_stage())
    elif "--check" in sys.argv:
        # ... existing check mode (still broken, but leave as-is for backward compat)
    else:
        # ... existing default mode
    ```

    Also fix the --check mode while we're here: currently both branches of the if/else do the same thing. Make --check actually compare: regenerate to a temp string, compare with existing file, exit 1 if different. This is separate from the hook (which auto-fixes), but makes --check useful for manual verification.

    Append to .pre-commit-config.yaml a new local repo entry:
    ```yaml
    - repo: local
      hooks:
        - id: sync-preprocessed-pyx
          name: Sync qint_preprocessed.pyx
          entry: python build_preprocessor.py --sync-and-stage
          language: system
          files: '(qint\.pyx|qint_\w+\.pxi)$'
          pass_filenames: false
    ```

    Per user decision: pre-commit framework (.pre-commit-config.yaml), not standalone script. Use `language: system` since build_preprocessor.py has no additional dependencies and runs in the user's environment.
