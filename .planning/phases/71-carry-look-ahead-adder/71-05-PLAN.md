Replace the NULL-returning `toffoli_QQ_add_bk()` stub with a working Brent-Kung carry-lookahead adder. The implementation uses the "compute-copy-uncompute" pattern with dedicated ancilla.

**Qubit layout for toffoli_QQ_add_bk(n):**
```
[0..n-1]              = register a (source, PRESERVED)
[n..2n-1]             = register b (target, gets a+b)
[2n..3n-2]            = generate ancilla g[0..n-2]
[3n-1..3n-2+tree_sz]  = tree propagate-product ancilla (BK internal)
[3n-1+tree_sz..end]   = carry-copy ancilla c[0..n-2]
```

**Add a static helper** `bk_cla_ancilla_count(int bits)` that computes the exact total ancilla count:
- n-1 generate ancilla
- tree_size propagate-product ancilla (depends on BK tree structure for given n)
- n-1 carry-copy ancilla
- Total: 2*(n-1) + tree_size

**BK tree structure computation:** For n-1 carry positions (0..n-2), the Brent-Kung tree has:
- Up-sweep: At level k (starting from 0), merge position 2^(k+1)-1 with position 2^(k+1)-1-2^k, then 2*2^(k+1)-1 with 2*2^(k+1)-1-2^k, etc. Only positions that exist (< n-1) are merged. Each merge at this level needs 1 tree propagate-product ancilla for the group propagate.
- Down-sweep: Fill in positions skipped by the up-sweep. Each fill needs 1 tree propagate-product ancilla for the group propagate used in the merge.

To simplify: compute tree_size by iterating through the BK tree levels and counting merges. The exact count depends on n. For small n: n=4 -> tree_size=1, n=8 -> tree_size=4, n=16 -> tree_size=11 (approximately).

**Algorithm (gate sequence):**

**Phase A: Initialize generate and propagate ancilla**
For i = 0 to n-2:
  1. CCX(target=g[i], ctrl1=a[i], ctrl2=b[i])  -- g[i] = a[i] AND b[i]
For i = 0 to n-1:
  2. CX(target=b[i], control=a[i])  -- b[i] = a[i] XOR b[i] = propagate p[i]

**Phase B: BK prefix tree forward sweep (up-sweep)**
For each level k = 0, 1, 2, ... up to ceil(log2(n-1))-1:
  stride = 2^(k+1)
  For each merge position j that is a multiple of stride minus 1, with partner j - 2^k:
    let i = j - 2^k (the left child)
    let prop_product = allocated tree ancilla for this merge
    3. CCX(target=g[j], ctrl1=p_src[j], ctrl2=g[i])  -- g[j] = g[j] OR (p[j] AND g[i])
       where p_src[j] is b[j] for first level (single-bit propagate), or the propagate-product ancilla from the previous level for this position
    4. CCX(target=prop_product, ctrl1=p_src[j], ctrl2=p_src[i])  -- prop_product = p[j]*p[i]
       This stores the group propagate for future levels.

**CRITICAL DETAIL on propagate products:** At level 0, the propagate source for position j is b[j]. At level k>0, the propagate source for position j is the propagate-product computed at level k-1 that covers the range ending at j. Track which ancilla holds the current group propagate for each tree position.

**Phase B2: BK prefix tree down-sweep**
For each level from ceil(log2(n-1))-2 down to 0:
  For each position that was NOT updated in the up-sweep but needs a carry:
    Find the nearest updated position to its left (call it parent).
    let prop_product = allocated tree ancilla for this down-sweep merge
    5. CCX(target=g[pos], ctrl1=b[pos_or_prop], ctrl2=g[parent])
    6. CCX(target=prop_product, ctrl1=b[pos_or_prop], ctrl2=p_of_parent)
    (Only if this position's group propagate is needed by further down-sweep merges)

**Phase C: Copy carries to carry-copy ancilla**
For i = 0 to n-2:
  7. CX(target=c[i], control=g[i])  -- c[i] = carry[i+1]

**Phase D: Reverse BK prefix tree (exact reverse of Phase B2 then B)**
Replay every gate from Phase B2 and B in reverse order. Since all tree operations are Toffoli/CNOT (self-inverse), replaying them in reverse uncomputes all tree modifications. After this:
- g[i] returns to single-bit generate (a[i] AND b[i])
- Tree propagate-product ancilla return to |0>
- b[i] still holds p[i] (tree only read b[] as controls, never modified it)

**Phase E: Uncompute propagates and generates**
For i = n-1 down to 0:
  8. CX(target=b[i], control=a[i])  -- restore b[i] to original
For i = n-2 down to 0:
  9. CCX(target=g[i], ctrl1=a[i], ctrl2=b[i])  -- uncompute g[i] (b[i] is original now)

After Phase E: g[] = all zeros, tree ancilla = all zeros, b[] = original values, a[] = unchanged.

**Phase F: Sum extraction**
For i = 0 to n-1:
  10. CX(target=b[i], control=a[i])  -- b[i] = p[i] = a[i] XOR b_orig[i]
For i = 1 to n-1:
  11. CX(target=b[i], control=c[i-1])  -- b[i] = p[i] XOR carry[i] = sum[i]

b[0] = p[0] = a[0] XOR b_orig[0] = sum[0] (no carry into bit 0).

After Phase F: b[] = a+b (sum), a[] = unchanged, g[] = |0>, tree ancilla = |0>, c[] = carry values (dirty, freed by allocator).

**IMPORTANT implementation notes:**
- The carry-copy ancilla c[] will NOT be uncomputed to |0>. They hold carry values. This is a known trade-off. Document clearly in the function comment.
- For bits < 2, return NULL (let dispatch use CDKM 1-bit special case).
- Use the existing `alloc_sequence()` helper to allocate the sequence.
- Compute the exact layer count before allocation: count all gates from phases A through F.
- Cache in `precompiled_toffoli_QQ_add_bk[bits]`.
- Use the existing `ccx()`, `cx()` gate emission functions.

**BK tree implementation strategy:** Rather than trying to compute exact BK merge positions dynamically, implement a helper that generates the list of (merge_pos, partner_pos, level) triples for a given n. Use arrays to track which tree ancilla holds the current group propagate for each position.

For the initial implementation, focus on getting widths 2-8 correct. Width 2 is trivial (1 carry position, no tree needed, just g[0] is the carry). Width 4 has 3 carry positions and a simple 2-level tree.

**Width 2 special case (1 carry position):**
Phase A: g[0] = a[0] AND b[0]; b[0..1] ^= a[0..1]
Phase B: (no tree -- only 1 carry position, g[0] is already carry[1])
Phase C: c[0] = g[0]
Phase D: (no tree to reverse)
Phase E: b[0..1] ^= a[0..1]; g[0] ^= a[0] AND b[0]
Phase F: b[0..1] ^= a[0..1]; b[1] ^= c[0]
Ancilla: 1 generate + 0 tree + 1 carry copy = 2 total

**Width 4 (3 carry positions, tree has 1 level):**
Phase A: g[0..2] = Toffoli; b[0..3] ^= a
Phase B up: g[1] ^= b[1] AND g[0]; tree_p0 = b[1] AND b[0]
Phase B down: g[2] ^= b[2] AND g[1]
Phase C: c[0..2] = g[0..2]
Phase D: reverse Phase B (3 Toffoli gates in reverse order)
Phase E: restore b, uncompute g
Phase F: propagate + carry -> sum
Ancilla: 3 generate + 1 tree + 3 carry copy = 7 total
