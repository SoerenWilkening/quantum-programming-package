**1. toffoli_CQ_add_bk(bits, value) -- BK CQ adder**

Follow the EXACT same temp-register pattern as `toffoli_CQ_add()` (lines 290-407 in ToffoliAddition.c):
1. Convert value to binary via `two_complement(value, bits)`
2. Count x_count (number of set bits)
3. Allocate sequence with layers: 2*x_count + bk_qq_layers (the layer count of BK QQ for this width)
4. Phase 1: X-init temp register -- for each set classical bit i, emit X(temp[i])
5. Phase 2: Run the BK QQ CLA algorithm core on temp[0..bits-1] as a-register and self[bits..2*bits-1] as b-register, using CLA ancilla at [2*bits..end]
   - IMPORTANT: Do NOT call toffoli_QQ_add_bk() and try to embed its sequence. Instead, inline the same gate emission logic that toffoli_QQ_add_bk() uses, but with the qubit indices offset for the CQ layout.
   - The CQ qubit layout is: [0..bits-1]=temp, [bits..2*bits-1]=self, [2*bits..end]=CLA ancilla. This maps directly: a-register=temp=[0..bits-1], b-register=self=[bits..2*bits-1], CLA ancilla=[2*bits..end].
6. Phase 3: X-cleanup temp register (same X gates as Phase 1)

Alternative (simpler): Since toffoli_QQ_add_bk() generates a cached sequence with qubit layout [0..n-1]=a, [n..2n-1]=b, [2n..end]=ancilla, and the CQ layout uses temp as a-register and self as b-register with the same relative positions, we can:
1. X-init temp
2. Get the QQ BK sequence via toffoli_QQ_add_bk(bits)
3. Inline-copy its gates with qubit offsets (or just use the same indices since CQ layout = temp[0..n-1] + self[n..2n-1] + ancilla[2n..end] which matches QQ layout exactly)
4. X-cleanup temp

Actually, the CQ approach can be EVEN simpler: build a sequence that is X-init + QQ-BK-gates + X-cleanup. The qubit layout matches perfectly because:
- CQ: [0..n-1]=temp (maps to a-register), [n..2n-1]=self (maps to b-register), [2n..]=CLA ancilla
- QQ: [0..n-1]=a, [n..2n-1]=b, [2n..]=CLA ancilla

So the CQ function can literally:
1. Get the QQ BK sequence pointer
2. Build a new sequence = X-init layers + QQ BK layers (copied) + X-cleanup layers
3. Return the new (owned, caller-frees) sequence

Use this approach for simplicity. Copy gates from the cached QQ BK sequence into the new CQ sequence. Use the same pattern as how toffoli_CQ_add copies the CDKM core.

For bits == 1: use existing CQ 1-bit logic (X gate or identity).

OWNERSHIP: Caller must free (NOT cached, value-dependent). Same as existing CQ add.

**2. toffoli_cQQ_add_bk(bits) -- Controlled BK CQ adder**

Follow the controlled CDKM pattern from `toffoli_cQQ_add()` (lines 409-481). For the controlled variant, every gate in the BK QQ sequence gets an additional control qubit:
- CCX(target, ctrl1, ctrl2) becomes MCX(target, [ctrl1, ctrl2, ext_ctrl]) -- 3-control gate
- CX(target, control) becomes CCX(target, control, ext_ctrl) -- 2-control gate

Qubit layout:
```
[0..n-1]              = a-register (source, preserved)
[n..2n-1]             = b-register (target, gets a+b when controlled)
[2n..2n+anc-1]        = CLA ancilla (same count as uncontrolled BK)
[2n+anc]              = ext_ctrl (external control qubit)
```

Implementation approach: Inline the BK CLA algorithm phases A-F (from plan 71-05), but replace every `ccx()` with `mcx(3 controls)` and every `cx()` with `ccx()` (adding ext_ctrl as additional control).

Use the same `emit_cMAJ`-style pattern: define `emit_controlled_ccx()` and `emit_controlled_cx()` helper macros or inline the gate emission.

OWNERSHIP: Cached in precompiled_toffoli_cQQ_add_bk[bits].

For bits == 1: single MCX(target=a[0], controls=[b[0], ext_ctrl]) = CCX. Wait -- for 1-bit controlled QQ add, the existing cQQ uses CCX(target=0, ctrl1=1, ctrl2=2). Keep the same pattern: return NULL for bits < 2 (let dispatch handle 1-bit via CDKM).

**3. toffoli_cCQ_add_bk(bits, value) -- Controlled BK CQ adder**

Follow the controlled CQ CDKM pattern from `toffoli_cCQ_add()` (lines 483-601):
1. CX-init temp register (controlled by ext_ctrl)
2. Controlled BK QQ core (same as cQQ BK, with temp as a-register)
3. CX-cleanup temp register (controlled by ext_ctrl)

Qubit layout:
```
[0..n-1]              = temp register (controlled init to classical value)
[n..2n-1]             = self register (target)
[2n..2n+anc-1]        = CLA ancilla
[2n+anc]              = ext_ctrl
```

Simplest approach: same as CQ but with controlled init/cleanup (CX instead of X, using ext_ctrl), and controlled BK core (same gates as cQQ BK).

OWNERSHIP: Caller must free (NOT cached, value-dependent).
