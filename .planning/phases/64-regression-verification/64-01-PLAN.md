  <name>Task 1: Run post-Phase-63 benchmarks and build comparison script</name>
  <files>
    benchmarks/results/bench_raw_post63.json
    scripts/benchmark_compare.py
  </files>
  <action>
**Step 1: Verify build is current.**
Run `python setup.py build_ext --inplace` from the project root to ensure the .so files reflect Phase 63 changes. Check .so modification timestamps are recent (after Phase 63 commits from 2026-02-08).

**Step 2: Run the existing benchmark suite.**
Execute `PYTHONPATH=src python scripts/benchmark_sequences.py --output benchmarks/results/bench_raw_post63.json` to collect BENCH-01 (import time), BENCH-02 (first-call generation for all 9 ops at widths 1-16), and BENCH-03 (cached dispatch w=8 vs w=17). This reuses the proven Phase 62 script as-is. Do NOT overwrite `benchmarks/results/bench_raw.json` -- that is the Phase 62 baseline.

Expected runtime: 5-7 minutes. The script spawns ~144 subprocesses for BENCH-02 plus 20 for BENCH-01.

**Step 3: Create `scripts/benchmark_compare.py`.**
Write a comparison script that:
1. Loads `benchmarks/results/bench_raw.json` (Phase 62 baseline) and `benchmarks/results/bench_raw_post63.json` (post-Phase-63)
2. Compares BENCH-01 (import time): compute percentage change of median, verdict PASS if change <= 15%
3. Compares BENCH-02 (first-call): for each operation and width, compute percentage change, flag any > 15% regressions
4. Compares BENCH-03 (cached dispatch): compare per-operation ns/call values, flag > 15% regressions
5. Compares .so file sizes: compute per-file and total size changes
6. Outputs structured JSON to `benchmarks/results/regression_report.json` with fields: `overall_verdict` ("PASS"/"FAIL"), `bench_01`, `bench_02`, `bench_03`, `so_sizes`, `tolerance_pct` (15.0)
7. Outputs human-readable markdown to `benchmarks/results/regression_report.md` with tables for each comparison section

The script should accept `--before` and `--after` paths (defaulting to `bench_raw.json` and `bench_raw_post63.json`), `--tolerance` (default 15.0), and `--output-dir` (default `benchmarks/results/`).

Key design:
- Overall verdict is PASS only if ALL sections pass
- A section passes if no metric shows regression beyond tolerance
- Improvements (negative change) are always PASS
- Missing data in either file is flagged as WARNING, not FAIL
- The 15% tolerance accounts for ~8% stdev observed in Phase 62

Use only stdlib (json, os, sys, argparse). No external dependencies.

3. Verify bench_raw_post63.json has same structure as bench_raw.json (same operation keys, same width keys)
