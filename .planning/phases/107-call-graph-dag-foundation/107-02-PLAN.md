CRITICAL: For nested call parent tracking, the push must happen BEFORE the function body executes (in capture path) and the node recording must happen AFTER. The approach:
- At top of `__call__`, if not top-level and building DAG: note the current parent from stack
- After result computed: add node to DAG with that parent
- For the CAPTURE path specifically: push a temporary context with the new node index BEFORE `self._func(*args, **kwargs)` executes, so nested compiled calls see this node as their parent. Then pop after.

Simpler approach that works:
- In `_capture_and_cache_both`, around the `self._func(*args, **kwargs)` call (line ~860), push DAG context with a placeholder, then after capture, update the node.
- In `_replay`, the function body is NOT re-executed, so nested calls are NOT re-triggered. Record only the replayed node.

Actually the cleanest approach: Record the node in `__call__` AFTER the result is computed. For nested tracking during capture, modify the capture path to push/pop DAG context around `self._func(*args, **kwargs)`:

def test_opt1_produces_dag():
    ql.circuit(4)
    @ql.compile(opt=1)
    def add_one(x):

        return x
