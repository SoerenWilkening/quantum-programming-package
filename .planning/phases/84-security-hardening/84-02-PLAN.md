  <name>Task 1: Run static analysis, fix findings, and create suppression file</name>
  <files>
    c_backend/suppressions.txt

    c_backend/include/*.h
  </files>
  <action>
    **Step 1: Install static analysis tools if not present**

    Check if `cppcheck` and `clang-tidy` are available:
    ```bash
    which cppcheck || echo "cppcheck not found"
    which clang-tidy || echo "clang-tidy not found"
    ```

    If not installed, install them:
    - On the current platform (Linux): `apt-get install -y cppcheck clang-tidy` (may need sudo)
    - If install not possible, document the commands needed and note that findings must be addressed when tools are available

    **Step 2: Create clang-tidy configuration**

    Create `c_backend/.clang-tidy` with C-relevant checks only (exclude C++-specific checks):
    ```yaml
    Checks: >
      bugprone-*,
      cert-*,
      clang-analyzer-*,
      misc-*,
      -misc-unused-parameters,
      -cert-dcl37-c,
      -cert-dcl51-cpp,
      -bugprone-easily-swappable-parameters,
      -clang-analyzer-security.insecureAPI.DeprecatedOrUnsafeBufferHandling
    HeaderFilterRegex: 'c_backend/include/.*'
    WarningsAsErrors: ''
    ```

    Rationale for exclusions:
    - `misc-unused-parameters`: Many callback/interface functions have unused params by design
    - `cert-dcl37-c` / `cert-dcl51-cpp`: Reserved identifier warnings for standard header patterns
    - `bugprone-easily-swappable-parameters`: Too noisy for C APIs with multiple int/qubit_t params
    - `insecureAPI.DeprecatedOrUnsafeBufferHandling`: memcpy/memset are intentional in this codebase

    **Step 3: Run cppcheck on all C backend files**

    Run cppcheck first (broader analysis):
    ```bash
    cppcheck --enable=all --inconclusive \
        --suppress=missingIncludeSystem \
        -I c_backend/include \
        --std=c11 \
        --template='{file}:{line}: [{severity}] {id}: {message}' \
        c_backend/src/ c_backend/include/ 2>&1 | tee /tmp/cppcheck-results.txt
    ```

    Note: Run on ALL files including `c_backend/src/sequences/` directory. The `--suppress=missingIncludeSystem` avoids noise from system headers.

    **Step 4: Run clang-tidy on non-sequence C files**

    Run clang-tidy on the main source files (not auto-generated sequences -- those are checked by cppcheck only):
    ```bash
    clang-tidy \
        --config-file=c_backend/.clang-tidy \
        c_backend/src/*.c \
        -- -I c_backend/include -std=c11 2>&1 | tee /tmp/clang-tidy-results.txt
    ```

    Note: Exclude `c_backend/src/sequences/` from clang-tidy (auto-generated code).

    **Step 5: Categorize and fix findings**

    For each finding:

    a. **If it's a real bug (undefined behavior, memory leak, null deref, buffer overrun):** FIX IT in the source code. These are the most important findings.

    b. **If it's a style/warning finding that can be improved:** FIX IT (user decided to fix ALL severities).

    c. **If it's a false positive (tool doesn't understand the context):** Add to `c_backend/suppressions.txt` with justification.

    Common expected findings and how to handle them:
    - **Unused function parameters:** Fix by casting to `(void)param;` or adding `__attribute__((unused))`
    - **Missing default in switch:** Add default case if appropriate
    - **Possible NULL dereference after malloc:** Already handled in most places (return NULL), verify remaining spots
    - **Variable scope reduction:** Move declarations closer to first use where safe
    - **Magic numbers:** Acceptable for gate/qubit constants -- suppress with justification if needed

    **Step 6: Create suppression file**

    Create `c_backend/suppressions.txt` with format:
    ```
    // Justification: [why this is a false positive or acceptable]
    [cppcheck-error-id]:[file]:[line]
    ```

    Each suppression MUST have a justification comment explaining why it's acceptable.

    **Step 7: Re-run analysis to verify clean results**

    After all fixes:
    ```bash
    cppcheck --enable=all --inconclusive \
        --suppressions-list=c_backend/suppressions.txt \
        --suppress=missingIncludeSystem \
        -I c_backend/include \
        --std=c11 \
        --error-exitcode=1 \
        c_backend/src/ c_backend/include/
    ```

    This should exit with code 0 (no unsuppressed findings).
  </action>
  <verify>
    1. `cppcheck --enable=all --suppressions-list=c_backend/suppressions.txt --suppress=missingIncludeSystem -I c_backend/include --std=c11 --error-exitcode=1 c_backend/src/ c_backend/include/` exits with code 0
    2. `clang-tidy --config-file=c_backend/.clang-tidy c_backend/src/*.c -- -I c_backend/include -std=c11` produces zero errors (warnings may remain if suppressed)
    3. `python setup.py build_ext --inplace` builds successfully after fixes
    4. `pytest tests/python/ -v` -- all tests pass (no regressions from fixes)
  </verify>
  <done>

    **Step 1: Add static analysis targets to Makefile**

    Add the following targets to the Makefile in the "Code Quality" section:

    ```makefile
    .PHONY: cppcheck
    cppcheck:
    	@echo "Running cppcheck static analysis on C backend..."
    	cppcheck --enable=all --inconclusive \
    		--suppressions-list=c_backend/suppressions.txt \
    		--suppress=missingIncludeSystem \
    		-I c_backend/include \
    		--std=c11 \
    		--error-exitcode=1 \
    		c_backend/src/ c_backend/include/
    	@echo "cppcheck: PASSED (zero findings)"

    .PHONY: clang-tidy
    clang-tidy:
    	@echo "Running clang-tidy on C backend..."
    	clang-tidy \
    		--config-file=c_backend/.clang-tidy \
    		c_backend/src/*.c \
    		-- -I c_backend/include -std=c11
    	@echo "clang-tidy: complete"

    .PHONY: static-analysis
    static-analysis: cppcheck clang-tidy
    	@echo "All static analysis checks complete."
    ```

    Also update the `help` target to list the new targets:
    ```
    @echo "  cppcheck        - Run cppcheck static analysis on C backend"
    @echo "  clang-tidy      - Run clang-tidy on C backend"
    @echo "  static-analysis - Run all static analysis (cppcheck + clang-tidy)"
    ```
