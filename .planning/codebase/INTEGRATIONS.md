# External Integrations

**Analysis Date:** 2026-02-22

## APIs & External Services

**Quantum Simulation:**
- Qiskit / qiskit-aer - Circuit simulation backend used to verify generated quantum circuits
  - SDK/Client: `qiskit>=1.0`, `qiskit_aer 0.17.2`
  - Usage: Lazy-imported (not a startup dependency); invoked inside `src/quantum_language/grover.py` and `src/quantum_language/amplitude_estimation.py` only when running end-to-end Grover search or IQAE
  - Auth: None (local simulation, no remote API calls)
  - Constraint: Max 17 qubits, `max_parallel_threads=4` enforced in all simulation calls

**Simulation Pattern:**
```python
# src/quantum_language/grover.py:258-266
import qiskit.qasm3
from qiskit import transpile
from qiskit_aer import AerSimulator

circuit_qk = qiskit.qasm3.loads(qasm_str)   # parse OpenQASM 3.0 string
sim = AerSimulator(max_parallel_threads=4)   # local statevector simulator
result = sim.run(transpile(circuit_qk, sim), shots=1).result()
```

**OpenQASM 3 Standard:**
- The project exports circuits to OpenQASM 3.0 format via `src/quantum_language/openqasm.pyx`
- `openqasm3 1.0.1` and `qiskit_qasm3_import 0.6.0` handle parsing on the Qiskit side
- No external API — format is self-contained string exchange between C backend and Qiskit

**Q# / Microsoft Quantum SDK (reference only):**
- `circuit-gen-results/qsharp/` contains a legacy C# host (`Program.cs`) and Q# source (`MyQFT.qs`) used for benchmarking QFT circuit generation
- Uses `Microsoft.Quantum.Sdk 0.28.302812` and `Microsoft.Quantum.Simulation.Simulators`
- Not part of the active build pipeline; `.gitignore` excludes compiled Q# artifacts
