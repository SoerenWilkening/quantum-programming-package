# External Integrations

**Analysis Date:** 2026-02-22

## APIs & External Services

**Quantum Circuit Simulation:**
- Qiskit Aer (`AerSimulator`) - Simulates exported OpenQASM 3.0 circuits to verify correctness and run Grover/IQAE algorithms.
  - SDK/Client: `qiskit_aer` 0.17.2; imported as `from qiskit_aer import AerSimulator`
  - Used in: `src/quantum_language/grover.py`, `src/quantum_language/amplitude_estimation.py`, `tests/conftest.py`
  - Always invoked with `AerSimulator(max_parallel_threads=4)` — hard constraint (see project memory)
  - For verification tests: `AerSimulator(method="statevector", max_parallel_threads=4)`, `shots=1`
  - For Grover/IQAE simulation: default method, configurable shots

**OpenQASM 3.0 Parsing:**
- Qiskit QASM3 - Loads OpenQASM 3.0 strings output by `ql.to_openqasm()` into Qiskit circuit objects.
  - SDK/Client: `qiskit.qasm3` module (part of `qiskit 2.3.0`)
  - Used in: `tests/conftest.py`, `src/quantum_language/grover.py`, `src/quantum_language/amplitude_estimation.py`
  - Call pattern: `qiskit.qasm3.loads(qasm_str)` followed by `circuit.measure_all()` if no classical registers

## Circuit Export Format

**Output:**
- OpenQASM 3.0 via `ql.to_openqasm()` implemented in `src/quantum_language/openqasm.pyx`
  - Calls C function `circuit_to_qasm_string()` from `c_backend/src/circuit_output.c`
  - Outputs `OPENQASM 3.0; include "stdgates.inc"; qubit[N] q; bit[N] c; ...`
  - Used as the integration point between the quantum language and Qiskit simulation

**Simulation Constraint (critical):**
- Max qubits for Qiskit/Aer simulation: **17 qubits**
- Max parallel threads: **4** (`max_parallel_threads=4` always set)
- These limits apply to all simulations (tests, demos, benchmarks, IQAE rounds)

## Statistical Dependencies

**IQAE Algorithm:**
- scipy.stats.beta - Clopper-Pearson exact binomial confidence intervals in `src/quantum_language/amplitude_estimation.py`
  - `from scipy.stats import beta`
  - `beta.ppf(alpha/2, ...)` — used for IQAE confidence interval computation
  - Required only when `ql.amplitude_estimate()` is called; scipy is an optional dependency at package level but required at runtime for this feature
