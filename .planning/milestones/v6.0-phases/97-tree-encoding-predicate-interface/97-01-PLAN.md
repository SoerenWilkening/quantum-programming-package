from qiskit import transpile
from qiskit_aer import AerSimulator

def _simulate_statevector(qasm_str):
    circuit = qiskit.qasm3.loads(qasm_str)
    circuit.save_statevector()
    sim = AerSimulator(method="statevector", max_parallel_threads=4)
    result = sim.run(transpile(circuit, sim)).result()
    return result.get_statevector()

Create `src/quantum_language/walk.py` with:

1. **TreeNode class** -- lightweight state wrapper for predicate evaluation:
   - Constructor takes `(height_register, branch_registers, max_depth)`
   - `.depth` property -- returns the one-hot height register as a qint (max_depth+1 qubits). Users compare with `node.depth == ql.qint(level, width=max_depth+1)`
   - `.branch_values` property -- returns list of qint branch registers (one per depth level)
   - Store `_max_depth` for later use by predicate interface (Plan 97-02)

2. **QWalkTree class** -- main tree encoding:
   - Constructor signature: `QWalkTree(max_depth, branching, predicate=None, *, max_qubits=17)`
   - Store `max_depth`, `max_qubits`, normalized `branching` list
   - **Branching normalization**: if `branching` is int, expand to `[branching] * max_depth`. If list, validate `len(branching) == max_depth`. Raise `ValueError` if neither.
   - **Register allocation** (eager, at construction):
     - Call `ql.circuit()` to ensure circuit exists (follow grover.py pattern)
     - Height register: `ql.qint(0, width=max_depth + 1)` -- one-hot encoding, max_depth+1 qubits
     - Branch registers: Python list of `ql.qint(0, width=max(1, math.ceil(math.log2(branching[i]))))` for each `i in range(max_depth)`
     - **CRITICAL**: Handle `branching[i] == 1` as special case: `math.log2(1) = 0`, so use `max(1, ...)` to ensure at least 1 qubit per branch register
   - **Root state preparation** (automatic, per CONTEXT.md -- no separate method):
     - Set height qubit for max_depth to |1> via `emit_x`
     - Height qubit index: `height_register.qubits[64 - (max_depth + 1) + max_depth]` which simplifies to `height_register.qubits[63]` (MSB of the allocated width)
     - All branch registers stay at |0> (default from qint(0, ...))
   - **Exposed attributes** (per CONTEXT.md):
     - `self.height_register` -- the one-hot height qint
     - `self.branch_registers` -- list of qint, one per depth level
     - `self.max_depth` -- integer
     - `self.branching` -- normalized list of branching degrees
     - `self.max_qubits` -- configurable simulation budget (default 17)
   - **total_qubits property**: computed as `(max_depth + 1) + sum(max(1, math.ceil(math.log2(b))) for b in self.branching)`
   - **TreeNode accessor**: `self.node` property returns `TreeNode(self.height_register, self.branch_registers, self.max_depth)` -- used by predicate interface in Plan 97-02
   - **Simulation budget check**: `_check_qubit_budget(self)` method that raises `ValueError` if `total_qubits > max_qubits`. This is called externally (by simulate/export methods in later phases), NOT at construction. Per CONTEXT.md: "Constructor allows any size tree to be created regardless of qubit count."
   - Store `self._predicate = predicate` for later use (Plan 97-02 will add validation). Do NOT call or validate predicate in this plan.

3. **Module docstring** explaining quantum backtracking tree encoding per Montanaro 2015.

4. **Update `src/quantum_language/__init__.py`**:
   - Add import: `from .walk import QWalkTree` (place after the quantum_counting import)
   - Add `"QWalkTree"` to the `__all__` list

Do NOT import or use qarray for branch registers -- use a plain Python list of qint as recommended by research (qarray adds overhead, and branch registers need independent per-level access).
