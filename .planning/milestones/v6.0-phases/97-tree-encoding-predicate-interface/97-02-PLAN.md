2. **Add `_validate_predicate` method** to QWalkTree:
   ```python
   def _validate_predicate(self):
       """Validate predicate mutual exclusion on root state.

       Calls the predicate on the current (root) state and checks that
       both is_accept and is_reject are not simultaneously True.
       Raises ValueError if mutual exclusion is violated.
       """
   ```
   Implementation:
   - Create TreeNode from current registers: `node = TreeNode(self.height_register, self.branch_registers, self.max_depth)`
   - Call predicate: `result = self._predicate(node)`
   - Validate result is a tuple/list of length 2: `if not isinstance(result, (tuple, list)) or len(result) != 2: raise TypeError("Predicate must return (is_accept, is_reject) tuple of two qbools")`
   - Extract `is_accept, is_reject = result`
   - Validate both are qbool (or have a value attribute): check `isinstance(is_accept, qbool) and isinstance(is_reject, qbool)`, raise TypeError if not
   - **Mutual exclusion check**: At construction, the root state is a computational basis state, so qbool values are deterministic. Check if both `.value` are truthy:
     ```python
     # Root is computational basis state -- classical check suffices
     accept_val = int(is_accept)  # qbool supports __int__
     reject_val = int(is_reject)
     if accept_val and reject_val:
         raise ValueError(
             "Predicate mutual exclusion violated: both accept and reject "
             "returned True for the root state. A node cannot be simultaneously "
             "accepted and rejected."
         )
     ```
   - Store the result qbools: `self._accept = is_accept`, `self._reject = is_reject` (for later use by walk operators)
   - **Compiled predicate support**: If `isinstance(self._predicate, CompiledFunc)`, store `self._predicate_is_compiled = True` for later uncomputation support. Otherwise `self._predicate_is_compiled = False`.

3. **Update constructor** to call validation when predicate is provided:
   - After register allocation and root state preparation (existing code from Plan 97-01)
   - Add: `if self._predicate is not None: self._validate_predicate()`
   - Store `self._predicate_is_compiled = False` as default before validation

4. **Add `uncompute_predicate` method** for predicate cleanup:
   ```python
   def uncompute_predicate(self):
       """Uncompute predicate qbools.

       For @ql.compile predicates, calls .inverse() to cleanly uncompute.
       For raw callables, this is a no-op (user manages cleanup).
       """
       if self._predicate is None:
           return
       if self._predicate_is_compiled:
           node = TreeNode(self.height_register, self.branch_registers, self.max_depth)
           self._predicate.inverse(node)
   ```

**IMPORTANT scope safety**: The predicate is called at the top level of the constructor (scope depth 0), NOT inside any `with` block. This ensures predicate-returned qbools are not registered in any LIFO scope stack and will not be auto-uncomputed. This matches the CONTEXT.md requirement for "raw allocation or @ql.compile inverse pattern."
