# Milestone v2.2: Performance Optimization

**Status:** SHIPPED 2026-02-08
**Phases:** 55-61
**Total Plans:** 22

## Overview

Profiling-driven performance optimization of the Quantum Assembly circuit generation pipeline. Established profiling infrastructure, then systematically optimized Cython hot paths, pre-computed gate sequences for common widths, migrated top hot paths to C, and eliminated memory allocation overhead.

## Milestone Summary

**Key Decisions:**
- Profile before optimizing — all phases driven by measured data
- Defer CYT-04 (nogil) from Phase 57 to Phase 60 due to Python call dependencies
- Two C entry points per operation (QQ and CQ variants) instead of single function
- Stack-allocated gate_t safe because add_gate() copies via memcpy
- Arena allocator unnecessary — remaining allocs are amortized realloc
- MEM-02 (inject_remapped_gates) was N/A — profiling showed different bottleneck

**Issues Resolved:**
- Forward/inverse depth parity confirmed and regression-tested
- Per-gate memory leak in run_instruction() eliminated
- #include ordering bug in generated sequence files fixed
- 32-bit multiplication segfault discovered (buffer overflow in C backend)

**Issues Deferred:**
- 32-bit multiplication segfault (buffer overflow) — needs investigation
- Pre-existing TODOs in qint.pyx (lines 610, 677) — not in hot paths
- Pre-existing TODO in optimizer.c line 27 (binary search improvement)
- Benchmark variability makes precise speedup measurement difficult

**Technical Debt Incurred:**
- Pre-existing segfault in test_array_creates_list_of_qint remains
- Benchmark 2-10x Cython improvement target not consistently measurable (system variability)
