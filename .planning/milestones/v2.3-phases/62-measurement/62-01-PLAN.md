  <action>
Create `scripts/benchmark_sequences.py` -- the primary benchmark measurement script. Create `benchmarks/results/` directory with a `.gitkeep`.

The script must implement three measurement sections:

**Section 1 -- BENCH-01: Import Time Measurement**
- Use `subprocess.run([sys.executable, "-c", ...])` to measure import time in a clean process
- The subprocess script: `import time; s=time.perf_counter_ns(); import quantum_language; print(time.perf_counter_ns()-s)`
- Run 20 iterations, collect all times
- Report median, mean, stdev in milliseconds
- Pass `env={**os.environ, "PYTHONPATH": os.path.join(project_root, "src")}` and `cwd=project_root`
- Also measure .so file sizes for all compiled extensions (`glob.glob("src/quantum_language/*.so")`) to estimate hardcoded sequence binary impact

**Section 2 -- BENCH-02: First-Call Generation Cost**
- For each of the 9 operations (QQ_add, CQ_add, cQQ_add, cCQ_add, QQ_mul, CQ_mul, Q_xor, Q_and, Q_or) at widths 1-16:
  - Use subprocess to get a clean process (global C caches cannot be reset)
  - The subprocess script creates a circuit, creates operands, times the FIRST call to the operation
  - Run 10 subprocess iterations per (operation, width) pair, report median
- Operation mapping (the subprocess inline Python code for each):
  - QQ_add: `a += b` where both are `ql.qint`
  - CQ_add: `a += 3` where a is `ql.qint`
  - cQQ_add: `ctrl = ql.qbool(True); ctrl.__enter__(); a += b; ctrl.__exit__(None,None,None)` (controlled context)
  - cCQ_add: `ctrl = ql.qbool(True); ctrl.__enter__(); a += 3; ctrl.__exit__(None,None,None)` (controlled context)
  - QQ_mul: `c = a * b` where both are `ql.qint` (cap at width 16, skip if width > 24 for safety per MEM-01)
  - CQ_mul: `c = a * 3` where a is `ql.qint`
  - Q_xor: `a ^= b` where both are `ql.qint`
  - Q_and: `c = a & b` where both are `ql.qint`
  - Q_or: `c = a | b` where both are `ql.qint`
- Print progress as it runs (e.g., "BENCH-02: QQ_add width=4 [10/144]") since this section takes ~5 minutes

**Section 3 -- BENCH-03: Cached Dispatch Overhead**
- For addition operations only (the only ones with both hardcoded and dynamic paths):
  - Measure cached dispatch at width 8 (hardcoded) vs width 17 (dynamic) for QQ_add and CQ_add
  - Use in-process measurement: create circuit, create operands, warm up (1 call), then time 50,000 iterations
  - Report per-call nanoseconds for each
- This runs in-process (cache is what we want to measure), not subprocess

**Output:**
- Print human-readable summary to stdout as each section completes
- Save all measurements to `benchmarks/results/bench_raw.json` with structure:
```json
{
  "bench_01_import_time": {
    "median_ms": ..., "mean_ms": ..., "stdev_ms": ..., "samples": ...,
    "so_file_sizes_bytes": {"qint.so": ..., ...}
  },
  "bench_02_first_call_us": {
    "QQ_add": {"1": ..., "2": ..., ..., "16": ...},
    "CQ_add": {"1": ..., ...},
    ...
  },
  "bench_03_cached_dispatch_ns": {
    "QQ_add": {"8_hardcoded": ..., "17_dynamic": ...},
    "CQ_add": {"8_hardcoded": ..., "17_dynamic": ...}
  },
  "metadata": {"python_version": ..., "timestamp": ..., "iterations": {...}}
}
```

**CLI interface:**
- Use argparse with `--bench` flag to select which benchmarks to run (default: all)
- `--bench 1` for import time only, `--bench 2` for first-call only, `--bench 3` for cached dispatch only, `--bench 1,2,3` for all
- `--iterations` to override default iteration counts
- `--widths` to override default width range (default: "1-16")

**Follow the existing pattern** from `scripts/profile_memory_61.py` for script structure (argparse, project_root detection, PYTHONPATH handling).

**IMPORTANT:** Use `time.perf_counter_ns()` (not `time.perf_counter()` or `time.time()`). Use `statistics.median()` and `statistics.stdev()` for summarization. Handle subprocess failures gracefully (skip and warn, do not crash).
