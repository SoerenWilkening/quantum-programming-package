- These tests will be slow (multi-shot, multiple rounds) -- use `@pytest.mark.slow` if needed

1. `test_known_single_solution_lambda`: Use `ql.amplitude_estimate(lambda x: x == 5, width=3, epsilon=0.05)`. True probability is 1/8 = 0.125. Assert `abs(result.estimate - 0.125) < 0.05` (within epsilon). Assert `result.num_oracle_calls > 0`. Assert `isinstance(result.estimate, float)`.

2. `test_known_multi_solution_lambda`: Use `ql.amplitude_estimate(lambda x: x > 5, width=3, epsilon=0.05)`. For 3-bit register, x > 5 means x in {6, 7}, so M=2, true probability = 2/8 = 0.25. Assert `abs(result.estimate - 0.25) < 0.05`.

3. `test_inequality_predicate_lambda`: Use `ql.amplitude_estimate(lambda x: x < 4, width=3, epsilon=0.05)`. x < 4 means x in {0,1,2,3}, so M=4, true probability = 4/8 = 0.5. Assert `abs(result.estimate - 0.5) < 0.05`.

4. `test_epsilon_affects_precision`: Run same oracle with epsilon=0.1 and epsilon=0.01. The tighter epsilon should use more oracle calls (`result2.num_oracle_calls >= result1.num_oracle_calls` in expectation -- use a generous tolerance since this is probabilistic).

5. `test_max_iterations_cap`: Use `ql.amplitude_estimate(lambda x: x == 3, width=3, epsilon=0.001, max_iterations=10)`. With very tight epsilon and very low max_iterations, should hit the cap. Use `pytest.warns(UserWarning)` to verify warning is emitted. Result should still have a valid `.estimate` float.

6. `test_result_is_float_like`: Run amplitude estimation, then verify `float(result)` works, `result + 0.0` works, `result > 0` works -- confirms AmplitudeEstimationResult integration in a real scenario.

7. `test_two_bit_register`: Use `ql.amplitude_estimate(lambda x: x == 1, width=2, epsilon=0.05)`. True probability = 1/4 = 0.25. Assert within epsilon. This tests minimal register size.

**Test structure notes:**

- Use `import math` for known probability calculations
- Each test should have a docstring explaining the expected probability and why
- For probabilistic tests, use generous tolerances (epsilon + 0.01 margin) since IQAE is statistical
- Keep register widths at 2-3 bits to stay well under 17-qubit limit with ancilla overhead
