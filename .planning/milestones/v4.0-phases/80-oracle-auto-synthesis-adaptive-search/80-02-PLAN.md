  <action>
Add BBHT adaptive search to grover.py and wire the adaptive/exact dispatch:

1. Add `import random` and `import warnings` at the top of grover.py.

2. Create `_run_grover_attempt(oracle, register_widths, j)` helper:
   - Creates a fresh circuit: `circuit()`, `option("fault_tolerant", True)`
   - Allocates registers: `qint_type(0, width=w) for w in register_widths`
   - Initializes superposition: `reg.branch(0.5)` for each register
   - Applies `j` Grover iterations (oracle + H-diffusion-H, same pattern as existing code)
   - Exports QASM and simulates single-shot: `_simulate_single_shot(to_openqasm(), register_widths)`
   - Returns tuple of measured values

3. Create `_bbht_search(oracle, register_widths, max_attempts, predicate=None)` function:
   - Implements BBHT algorithm (Boyer-Brassard-Hoyer-Tapp, 1998)
   - Growth factor `LAMBDA = 6/5` (from BBHT paper)
   - `N = product(2**w for w in register_widths)`; `sqrt_N = math.sqrt(N)`
   - `default_max_attempts = math.ceil(2 * math.log2(N))` if max_attempts is None
   - For each attempt m in range(max_attempts):
     - `upper = min(LAMBDA ** m, sqrt_N)`
     - `j = random.randint(0, max(1, int(upper)) - 1)` -- random in [0, upper)
     - Call `_run_grover_attempt(oracle, register_widths, j)` to get measured values
     - Track `total_iterations += j`
     - Classical verification: if predicate is not None, call `_verify_classically(predicate, values)`. If valid, return `(*values, total_iterations)`.
     - If no predicate (decorated oracle), return `(*values, total_iterations)` on first measurement (no verification possible).
   - If exhausted all attempts without finding valid solution, return None
   - Emit `warnings.warn` if predicate + register width sum exceeds 14 qubits (approaching 17-qubit simulator limit considering ancilla overhead)

4. Update `grover()` dispatch logic -- replace the Plan 01 placeholder:
   - Remove the `m = 1` placeholder that Plan 01 added for `m is None`
   - When `m is not None`: use exact iteration count path (existing logic, refactored into `_run_grover_attempt` for a single shot with `k = iterations or _grover_iterations(N, m)`)
   - When `m is None`: call `_bbht_search(oracle, register_widths, max_attempts, predicate)`
   - For the exact path (`m is not None`): refactor existing circuit-building logic to use `_run_grover_attempt(oracle, register_widths, k)` then return `(*values, k)`

5. Ensure `_run_grover_attempt` handles the oracle correctly:
   - For predicate oracles, `_predicate_to_oracle` was called in Plan 01 to create a GroverOracle
   - For decorated oracles, `_ensure_oracle` wraps as before
   - Both paths produce GroverOracle instances that `_run_grover_attempt` calls identically
   - IMPORTANT: Each attempt builds a fresh `circuit()` -- this is required because circuit state is not resettable mid-search. The GroverOracle cache may need clearing between attempts -- test this and handle if needed.

6. Handle the return format:
   - Exact path: `(*values, k)` tuple (same as before)
   - Adaptive path found solution: `(*values, total_iterations)` tuple
   - Adaptive path exhausted: `None`
   - Per CONTEXT.md: "Same return format: (value, iterations) regardless of adaptive or not"

IMPORTANT constraints:
- Max 17 qubits for simulation, max 4 threads for AerSimulator (already in _simulate_single_shot)
- Cap `j` at `sqrt(N)` to prevent Grover overrotation
- Do NOT modify `_simulate_single_shot` -- it already handles the 4-thread limit
- Each BBHT attempt must call `circuit()` for a fresh circuit -- stale circuit state between attempts causes errors
