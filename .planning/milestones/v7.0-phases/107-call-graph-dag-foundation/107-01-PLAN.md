  <action>
Create `src/quantum_language/call_graph.py` with the following components:

**DAGNode class** (use `__slots__` for performance):
- Attributes: `func_name` (str), `qubit_set` (frozenset of int), `gate_count` (int), `cache_key` (tuple), `bitmask` (np.uint64)
- Constructor computes bitmask from qubit_set: `bitmask = sum(1 << q for q in qubit_set)` stored as np.uint64

**_popcount_array helper** (module-level):
- Vectorized popcount using byte LUT (pre-compute LUT at module level as `_POPCOUNT_LUT`)
- Takes np.ndarray of uint64, returns np.ndarray of int32 with popcount per element
- Implementation: iterate over 8 byte positions, index into LUT, accumulate

**CallGraphDAG class**:
- `__init__`: Create `rx.PyDAG()`, empty `_nodes` list (node_index -> DAGNode)
- `add_node(func_name, qubit_set, gate_count, cache_key, parent_index=None)`: Create DAGNode, add to PyDAG, if parent_index is not None add edge with data `{'type': 'call'}`. Return node index.
- `build_overlap_edges()`: For all node pairs (i < j), compute bitmask AND + popcount. If weight > 0, add edge with data `{'type': 'overlap', 'weight': int(w)}`. Skip pairs that already have a 'call' edge (parent-child). Use vectorized NumPy approach from research.
- `parallel_groups()`: Build undirected `rx.PyGraph` with same nodes. Add edges for all pairs with overlapping qubits. Return `rx.connected_components(g)` as list of sets.
- `nodes` property: Return list of DAGNode objects.
- `node_count` property: Return `len(self._nodes)`.
- `__len__`: Return node count.
- `dag` property: Return underlying `rx.PyDAG` (for advanced queries by downstream code).

**Builder stack** (module-level, mirrors `_capture_depth` pattern):
- `_dag_builder_stack = []` -- stack of `(CallGraphDAG, parent_node_index_or_None)` tuples
- `push_dag_context(dag, parent_index=None)`: Append to stack
- `pop_dag_context()`: Pop from stack, return the tuple
- `current_dag_context()`: Return top of stack or None if empty

Write tests FIRST in `tests/python/test_call_graph.py`, then implement to make them pass. Tests should import directly from `quantum_language.call_graph` and test the module in isolation (no circuit or compile needed).
