  <action>
Add `_apply_merge(self)` method to `_CompiledFunc` class. This method:

1. Gets merge groups from `self._call_graph.merge_groups(self._merge_threshold)`
2. For each multi-node group:
   a. Collect (CompiledBlock, virtual_to_real) pairs from `self._cache` for each node's `cache_key`. Access blocks via `self._cache[node.cache_key]` and v2r via `block._capture_virtual_to_real`.
   b. Call `_merge_and_optimize(blocks_with_mappings, optimize=self._optimize)` to get merged gate list
   c. Create a merged `CompiledBlock` storing the optimized physical gates. This merged block uses an identity virtual-to-real mapping (gates are already in physical space). Set `total_virtual_qubits` to max physical qubit + 1, `param_qubit_ranges=[]`, `internal_qubit_count=0`, `return_qubit_range=None`.
   d. Store in `self._merged_blocks` dict keyed by frozenset of node indices in the group
3. If debug mode, store merge stats on `self._stats`

Wire into `__call__`:
- After `build_overlap_edges()` in the finally block, add:
```python
if self._opt == 2 and self._call_graph is not None:
    self._apply_merge()
```

For merged replay on subsequent calls:
- In `_call_inner`, when `is_hit` is True and `self._opt == 2` and `self._merged_blocks` is not None:
  - Check if the current node's cache_key belongs to a merge group
  - If yes, find the merged block and replay it using `inject_remapped_gates(merged_block.gates, {i: i for i in range(merged_block.total_virtual_qubits)})` (identity mapping since gates are already physical)
  - Only replay the merged block ONCE per group (first node in group triggers replay, subsequent nodes in same group are skipped since their gates are already included in the merged block)
  - This requires tracking which merge groups have been replayed in the current call via a set on the instance

IMPORTANT: On first call, gates are already in the circuit from normal capture. The merged blocks are for SUBSEQUENT call replays only. Do NOT double-inject on first call.

For controlled context support:
- When replaying a merged block inside a controlled context (`is_controlled=True`), derive controlled gates from the merged gate list using `_derive_controlled_gates()` on each gate (prepend control qubit to controls list). The control qubit is obtained from `_get_control_bool()`.

For cache reset:
- Clear `self._merged_blocks = None` in `_reset_for_circuit()` so merged cache is invalidated on circuit reset.
