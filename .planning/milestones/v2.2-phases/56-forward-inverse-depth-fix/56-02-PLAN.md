Based on the diagnostic results from Plan 01, implement the fix.

**Most Likely Fix (if layer_floor is the cause):**

The issue is that during replay, `_set_layer_floor(start_layer)` prevents gates from being packed into earlier layers. However, during capture, gates flow through add_gate() which can place them in earlier layers if qubits are available.

**Option A: Match capture behavior (preferred if capture produces optimal depth)**
- Don't set layer_floor during replay
- Let gates pack naturally via add_gate()
- Risk: Gates might reorder into earlier layers, affecting subsequent operations

**Option B: Match replay behavior for capture (if replay produces optimal depth)**
- Set layer_floor during capture as well
- Ensures consistent behavior but may increase capture depth

**Option C: Store and replay layer assignments (most complex)**
- During capture, record which layer each gate was assigned to
- During replay, explicitly place gates at the same relative layers
- Most precise but requires schema change

**Implementation approach:**

1. First, verify which path produces the better (lower) depth from Plan 01 diagnostics
2. If capture > replay depth: The fix is in capture path
3. If replay > capture depth: The fix is in replay path (most likely)

**If replay is inflating depth (expected case):**

In `_replay()` method (around line 984-994), the current code:
```python
saved_floor = _get_layer_floor()
start_layer = get_current_layer()
_set_layer_floor(start_layer)
inject_remapped_gates(block.gates, virtual_to_real)
end_layer = get_current_layer()
_set_layer_floor(saved_floor)
```

The fix depends on whether layer_floor is actually needed:

1. If layer_floor is for correctness (preventing gate reordering into pre-function layers), we need a different approach:
   - Calculate the expected depth from the cached gate sequence
   - Compare to actual depth after replay
   - If they match, no fix needed (issue is elsewhere)

2. If layer_floor is causing inflation:
   - Consider whether we can avoid setting it
   - Or set it to a value that doesn't inflate depth unnecessarily
