**Follow exact same pattern as add_seq_1_4.c** but for larger widths.

**Gate counts for reference (QQ_add):**
- Width 5: 45 gates, 23 layers (5*5-2)
- Width 6: 63 gates, 28 layers (5*6-2)
- Width 7: 84 gates, 33 layers (5*7-2)
- Width 8: 108 gates, 38 layers (5*8-2)

**Gate counts for cQQ_add are much higher due to CNOT + half-rotation structure.**
- Refer to cQQ_add() layer formula: `bits * (bits + 1) / 2 * 4 + 4 * bits - 2 - bits / 4 * 4 + 3`

**Static dispatch helpers for 5-8:**
```c
static const sequence_t* get_hardcoded_QQ_add_5_8(int bits) {
    switch (bits) {
        case 5: return &HARDCODED_QQ_ADD_5;
        case 6: return &HARDCODED_QQ_ADD_6;
        case 7: return &HARDCODED_QQ_ADD_7;
        case 8: return &HARDCODED_QQ_ADD_8;
        default: return NULL;
    }
}

static const sequence_t* get_hardcoded_cQQ_add_5_8(int bits) {
    switch (bits) {
        case 5: return &HARDCODED_cQQ_ADD_5;
        case 6: return &HARDCODED_cQQ_ADD_6;
        case 7: return &HARDCODED_cQQ_ADD_7;
        case 8: return &HARDCODED_cQQ_ADD_8;
        default: return NULL;
    }
}
```

**UNIFIED PUBLIC dispatch functions (at end of file):**

These are the entry points called from IntegerAddition.c. They call into _1_4 helpers (from add_seq_1_4.c) and _5_8 helpers (local).

```c
// Forward declarations for _1_4 helpers (implemented in add_seq_1_4.c)
extern const sequence_t* get_hardcoded_QQ_add_1_4(int bits);
extern const sequence_t* get_hardcoded_cQQ_add_1_4(int bits);

// PUBLIC unified dispatch - covers all 1-8
const sequence_t* get_hardcoded_QQ_add(int bits) {
    if (bits >= 1 && bits <= 4) {
        return get_hardcoded_QQ_add_1_4(bits);
    } else if (bits >= 5 && bits <= 8) {
        return get_hardcoded_QQ_add_5_8(bits);
    }
    return NULL;  // > 8 or < 1: caller must use dynamic generation
}

const sequence_t* get_hardcoded_cQQ_add(int bits) {
    if (bits >= 1 && bits <= 4) {
        return get_hardcoded_cQQ_add_1_4(bits);
    } else if (bits >= 5 && bits <= 8) {
        return get_hardcoded_cQQ_add_5_8(bits);
    }
    return NULL;
}
```

**Dispatch strategy clarification:**
- Plan 01 creates `get_hardcoded_QQ_add_1_4()` and `get_hardcoded_cQQ_add_1_4()` as static helpers
- This plan creates `_5_8()` variants as static helpers
- This plan implements the PUBLIC `get_hardcoded_QQ_add()` and `get_hardcoded_cQQ_add()` unified dispatch
- Unified dispatch handles routing to correct helper based on width

sequence_t *QQ_add(int bits) {
    // Bounds check: valid widths are 1-64
    if (bits < 1 || bits > 64) {
        return NULL;
    }

    // Use hardcoded sequences for widths 1-8
    if (bits <= HARDCODED_MAX_WIDTH) {
        const sequence_t* hardcoded = get_hardcoded_QQ_add(bits);

            // 1. Static sequences have program lifetime (never freed)
            // 2. Caller should never free returned sequences anyway

            return (sequence_t*)hardcoded;
        }
    }

}

**Modify cQQ_add() function (around line 334):**

Same pattern - add hardcoded check after bounds check, before cache check:
```c
sequence_t *cQQ_add(int bits) {
    // Bounds check: valid widths are 1-64
    if (bits < 1 || bits > 64) {
        return NULL;
    }

    // Use hardcoded sequences for widths 1-8
    if (bits <= HARDCODED_MAX_WIDTH) {
        const sequence_t* hardcoded = get_hardcoded_cQQ_add(bits);
        if (hardcoded != NULL) {
            // SAFETY: Const cast is safe here because:
            // 1. Static sequences have program lifetime (never freed)
            // 2. Caller should never free returned sequences anyway
            // 3. Cast needed only because existing API returns non-const
            return (sequence_t*)hardcoded;
        }
    }

    // Check cache for dynamically generated (widths > 8)
    if (precompiled_cQQ_add_width[bits] != NULL)
        return precompiled_cQQ_add_width[bits];

    // ... rest of dynamic generation unchanged ...
}
```

**Const casting rationale and safety:**
The existing QQ_add() and cQQ_add() API returns `sequence_t*` (non-const). The hardcoded sequences are `const sequence_t*` because they're static and should never be modified. We cast away const for API compatibility because:
1. **Lifetime safety:** Static sequences live for program duration - no memory management issues
2. **Caller contract:** Callers should NEVER free or modify returned sequences (existing contract)
3. **API compatibility:** Changing API to return const would be breaking change
4. **Practical safety:** No code path modifies returned sequences

**Do NOT modify:**
- CQ_add or cCQ_add (these use classical value parameters, not fully precomputable)
- Any other functions in the file
- The dynamic generation code paths
