  <name>Task 2: Implement QQ_add sequences for widths 1-4</name>
  <files>c_backend/src/sequences/add_seq_1_4.c</files>
  <action>
Create `c_backend/src/sequences/add_seq_1_4.c` implementing hardcoded QQ_add for widths 1-4.

**Structure pattern for each width N:**

```c
// --- Width N QQ_add ---
// Layer arrays: static const gate_t QQ_ADD_N_L0[], QQ_ADD_N_L1[], etc.
// Layer pointer array: static gate_t* QQ_ADD_N_LAYERS[]
// Gates per layer: static num_t QQ_ADD_N_GPL[]
// Wrapper sequence_t: static sequence_t HARDCODED_QQ_ADD_N
```

**Gate sequence reference from IntegerAddition.c and gate.c:**

For QQ_add(bits):
- Layers 0 to 2*bits-2: QFT (H gates + CP gates, MSB-first processing)
- Layers 2*bits-1 to 3*bits-2: Addition rotations (CP from control register)
- Layers 3*bits-1 to 5*bits-3: IQFT (reverse of QFT)

**Key formulas:**
- Total layers for QQ_add: `5 * bits - 2`
- QFT layers: `2 * bits - 1`
- Addition layers: `bits`
- IQFT layers: `2 * bits - 1`

**Qubit indices (canonical):**
- Target operand: qubits [0, bits-1], where 0 = LSB, bits-1 = MSB
- Control operand (b): qubits [bits, 2*bits-1], where bits = LSB, 2*bits-1 = MSB

**Rotation angles:** Use exact M_PI expressions: M_PI, M_PI/2, M_PI/4, M_PI/8, etc.

**gate_t initialization:** Use designated initializers with all fields:
```c
{.Gate = H, .Target = 0, .NumControls = 0, .Control = {0}, .large_control = NULL, .GateValue = 0, .NumBasisGates = 0}
{.Gate = P, .Target = 0, .NumControls = 1, .Control = {1}, .large_control = NULL, .GateValue = M_PI, .NumBasisGates = 0}
```

**Static dispatch helper at end of file:**
```c
const sequence_t* get_hardcoded_QQ_add_1_4(int bits) {
    switch (bits) {
        case 1: return &HARDCODED_QQ_ADD_1;
        case 2: return &HARDCODED_QQ_ADD_2;
        case 3: return &HARDCODED_QQ_ADD_3;
        case 4: return &HARDCODED_QQ_ADD_4;
        default: return NULL;
    }
}
```

**This task implements QQ_add only.** cQQ_add is significantly more complex and is implemented in Task 3.

<task type="auto">
  <name>Task 3: Implement cQQ_add sequences for widths 1-4</name>
  <files>c_backend/src/sequences/add_seq_1_4.c</files>
  <action>
Add hardcoded cQQ_add sequences for widths 1-4 to `add_seq_1_4.c`.

**cQQ_add structure is MORE COMPLEX than QQ_add:**
- Refer to cQQ_add() in IntegerAddition.c lines 334-448
- Uses half-rotations and CNOT gates
- Total layers formula: `bits * (bits + 1) / 2 * 4 + 4 * bits - 2 - bits / 4 * 4 + 3`
- Has 3 distinct blocks:
  1. Unconditional half-rotations (applied to all)
  2. CNOT + negative half-rotations + CNOT (controlled toggle)
  3. Controlled rotations from b register
- Control qubit at position `3 * bits - 1`

**Layer counts for cQQ_add:**
- Width 1: ~13 layers
- Width 2: ~25 layers
- Width 3: ~41 layers
- Width 4: ~61 layers (approximate - trace exact from code)

**Approach for deriving exact sequences:**
1. Trace through cQQ_add() dynamically for each width
2. Record each gate's type, target, controls, and angle
3. Group into layers as the dynamic code does
4. Transcribe to static arrays

**Add static dispatch helper:**
```c
const sequence_t* get_hardcoded_cQQ_add_1_4(int bits) {
    switch (bits) {
        case 1: return &HARDCODED_cQQ_ADD_1;
        case 2: return &HARDCODED_cQQ_ADD_2;
        case 3: return &HARDCODED_cQQ_ADD_3;
        case 4: return &HARDCODED_cQQ_ADD_4;
        default: return NULL;
    }
}
```

**Important:** Due to complexity, you may need to use the Python bindings to execute cQQ_add dynamically and extract the gate sequences, then manually convert to C static arrays. The cQQ_add formulas are complex enough that manual transcription is error-prone.
